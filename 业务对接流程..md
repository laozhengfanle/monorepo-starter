# 业务对接流程指南（Fork 后如何接入新业务）

> 适用对象：fork MonoKit 基座项目后，在其上构建新业务的开发者。
> 目标读者：第一次接触本基座、想新增一套完整业务模块（数据库表 + 后端 GraphQL + 前端页面 + 菜单/权限）的工程师。
> 案例：以"卖 XX 产品"商城为例，新增**产品分类**（ProductCategory）与**产品列表**（ProductItem）两个业务模块。
>
> **强制约束**（与基座项目级规则一致，违反会被门禁拦截）：
>
> 1. **API 一律 GraphQL** —— 业务模块**禁止**新增 RESTful 端点（RESTful 仅用于 BFF 层的认证、上传、健康检查等命令式操作）
> 2. **后台 UI 严格遵循** [开发文档/布局与UI规范.md](./开发文档/布局与UI规范.md) —— 禁止手搓 `<div class="p-6 ...">` 布局
> 3. **所有命名严格遵循** [开发文档/命名规范.md](./开发文档/命名规范.md) —— 字段名、文件名、类名、权限码一字不差
> 4. **完整规范参考** [开发文档/](./开发文档/) 与 [用户指南/](./用户指南/)

## 0. 总原则（先把规矩立好）

| #   | 原则                                                                                                                                                                                       |
| --- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 0.1 | **业务模块 100% GraphQL**。任何业务 CRUD（增删改查、关联查询）一律走 `@Query` / `@Mutation`；REST 端点（`bff/` 下）只允许在认证、文件上传、健康检查等命令式场景出现。                       |
| 0.2 | **入参校验 Zod 化**。所有 GraphQL `InputType` 必须有对应的 Zod schema（`packages/shared/src/schemas/<module>/`），通过 `@Args(name, { type: () => XxxInputType, nullable: false }, new ZodArgsPipe(Schema))` 三参数装饰器校验入参；**禁止**在 resolver 里写内联校验。       |
| 0.3 | **后端是菜单/权限的唯一来源**。前端不写静态路由文件，路由由后端 `admin_menu` 表的树 + GraphQL `getCurrentUserMenus` 动态生成；前端只负责把 `component` 字符串映射到具体 `.vue` 文件。       |
| 0.4 | **菜单分三类**：`directory`（目录，分组用）/ `menu`（页面，挂在 directory 下）/ `button`（按钮权限，挂在 menu 下）。三者组合成完整权限树。                                                  |
| 0.5 | **权限码格式**：`<资源>:<动作>[:<细分>]`，全部小写、冒号分隔。**操作词固定为** `list` / `create` / `update` / `delete` / `export` / `view` / `import`（见 [命名规范 §四](./开发文档/命名规范.md#四权限标识)）。例：`product:category:view`、`product:item:create`。 |
| 0.6 | **数据库表必须有审计字段**：`id`（UUID v7）+ `createdAt` + `updatedAt` + `createdBy` + `updatedBy` + `deletedAt`（软删）。**DB 字段 snake_case、TS 字段 camelCase（`@map`）**。               |
| 0.7 | **每个 Query/Mutation 必须挂权限码**（`@Permission('xxx')`），后端用 `AdminPermissionGuard` 强制校验；前端用 `v-permission` 指令控制按钮可见性。**前后端双层防御缺一不可**。                |
| 0.8 | **前端 UI 必遵循布局与 UI 规范**：列表页 = `<n-card title="...">` + `#header-extra` 头部按钮 + `<n-data-table>` + `<n-modal preset="card">` 表单；**禁止** `<div class="p-6 space-y-4">` 手搓布局。 |
| 0.9 | **表单校验**单一数据源：Zod schema → `zodToRules` → `zodToFormRules` / `zodToPopoverRules`，**禁止**硬编码 formRules（见 [布局与UI规范 §8.1](./开发文档/布局与UI规范.md#八表单校验规范)）。     |
| 0.10 | **测试与门禁**：typecheck / lint / test / build 四道门禁全过后才能 commit；本任务涉及的每一个新文件都按既有规范配 `.spec.ts` 单元测试。                                                  |

## 1. 业务案例拆解

**卖 XX 产品的商城**需要 2 个新模块：

| 模块         | 资源               | 主要功能                         | 路径前缀             |
| ------------ | ------------------ | -------------------------------- | -------------------- |
| **产品分类** | `product-category` | 树形分类管理（增删改查 + 上下级） | `/product/category`  |
| **产品列表** | `product-item`     | 产品 CRUD，关联分类、上下架      | `/product/item`      |

**菜单树设计**（最终在侧边栏的样子）：

```
产品管理 (directory)                ← /product
├── 分类管理 (menu)                 ← /product/category  →  ProductCategoryPage.vue
│   ├── 新增分类 (button)           ← product:category:create
│   ├── 编辑分类 (button)           ← product:category:update
│   └── 删除分类 (button)           ← product:category:delete
└── 产品列表 (menu)                 ← /product/item      →  ProductItemPage.vue
    ├── 新增产品 (button)           ← product:item:create
    ├── 编辑产品 (button)           ← product:item:update
    ├── 删除产品 (button)           ← product:item:delete
    └── 上下架 (button)             ← product:item:update  ← 见下方说明
```

> **关于"上下架"权限码**：命名规范 [§四](./开发文档/命名规范.md#四权限标识) 规定操作词只能从 `list / create / update / delete / export / view / import` 选。上下架本质是更新 `onSale` 字段，因此**复用 `product:item:update`**；若要更细颗粒度，可加 `:on-sale` 细分（`product:item:update:on-sale`），但仅在确实需要单独授权时才这么做。

**权限码全集**（后端 Guard 与前端按钮公用）：

```
product:category:view     — 分类管理页面（Query 列表/详情）
product:category:create   — 新增分类
product:category:update   — 编辑分类
product:category:delete   — 删除分类
product:item:view         — 产品列表页面（Query 列表/详情）
product:item:create       — 新增产品
product:item:update       — 编辑产品（含上下架）
product:item:delete       — 删除产品
```

## 2. 步骤一：数据库设计（Prisma schema）

### 2.1 设计原则（对齐 [命名规范 §一](./开发文档/命名规范.md#一数据库)）

- **表名**：snake_case **单数**（`product_category` / `product_item`），**不加 `s` 后缀**
- **列名**：DB 蛇形 + `@map(...)`、TS 驼峰（命名规范明确禁止驼峰进 DB）
- **主键**：`String @id @db.Uuid`，由 `uuidv7()`（[apps/server/src/common/utils/uuid.ts](./apps/server/src/common/utils/uuid.ts)）生成
- **必带 6 个审计字段**：`id` / `createdAt` / `updatedAt` / `createdBy` / `updatedBy` / `deletedAt`
- **树形结构**用自引用 `parentId`（参考 `AdminMenu.parentId`）
- **外键**用 `@relation(... onDelete: Restrict)`，**禁止 Cascade**（防误删，命名规范 §一默认 Restrict）
- **布尔列**用语义化名称（`enabled` / `visible` / `onSale`），**禁止 `is_` 前缀**

### 2.2 在 `apps/server/prisma/schema.prisma` 追加两个 model

打开文件，在末尾追加：

```prisma
// ============================================================
// 业务：产品分类（树形）
// ============================================================
/// 产品分类表 — 树形结构，自引用 parentId
/// - 通过 parentId 实现无限层级
/// - 排序：sort 升序
/// - 软删：deletedAt 不为空视为已删除
model ProductCategory {
  id          String   @id @db.Uuid
  parentId    String?  @map("parent_id") @db.Uuid
  name        String   @db.VarChar(100)
  /// 分类编码（URL slug 友好，全局唯一）
  code        String   @unique @db.VarChar(50)
  /// 图标（前端 Iconify 名称）
  icon        String   @default("") @db.VarChar(100)
  sort        Int      @default(0)
  enabled     Boolean  @default(true)
  createdAt   DateTime @default(now()) @map("created_at") @db.Timestamptz
  updatedAt   DateTime @updatedAt @map("updated_at") @db.Timestamptz
  createdBy   String?  @map("created_by") @db.Uuid
  updatedBy   String?  @map("updated_by") @db.Uuid
  deletedAt   DateTime? @map("deleted_at") @db.Timestamptz

  parent   ProductCategory?  @relation("ProductCategoryTree", fields: [parentId], references: [id], onDelete: Restrict)
  children ProductCategory[] @relation("ProductCategoryTree")
  items    ProductItem[]

  @@index([parentId])
  @@index([deletedAt])
  @@map("product_category")
}

// ============================================================
// 业务：产品
// ============================================================
/// 产品表 — 关联一个分类
/// - 上架状态：onSale=true 上架、false 下架（由 product:item:update 控制）
/// - 价格用 Decimal(10,2) 避免浮点误差
model ProductItem {
  id          String   @id @db.Uuid
  categoryId  String   @map("category_id") @db.Uuid
  name        String   @db.VarChar(200)
  /// 产品编码（业务主键，对外显示）
  spu         String   @unique @db.VarChar(50)
  /// 销售价
  price       Decimal  @db.Decimal(10, 2)
  /// 库存
  stock       Int      @default(0)
  /// 详情（可选，HTML/Markdown）
  description String   @default("") @db.Text
  /// 封面图 URL（相对路径，由上传服务生成）
  coverImage  String   @default("") @map("cover_image") @db.VarChar(500)
  onSale      Boolean  @default(false) @map("on_sale")
  enabled     Boolean  @default(true)
  createdAt   DateTime @default(now()) @map("created_at") @db.Timestamptz
  updatedAt   DateTime @updatedAt @map("updated_at") @db.Timestamptz
  createdBy   String?  @map("created_by") @db.Uuid
  updatedBy   String?  @map("updated_by") @db.Uuid
  deletedAt   DateTime? @map("deleted_at") @db.Timestamptz

  category ProductCategory @relation(fields: [categoryId], references: [id], onDelete: Restrict)

  @@index([categoryId])
  @@index([onSale])
  @@index([deletedAt])
  @@map("product_item")
}
```

### 2.4 步骤一约束清单

- ❌ **禁止**使用 `BigInt` 主键（基座统一 UUID v7）
- ❌ **禁止**用 `onDelete: Cascade`（基座默认 Restrict，防止误删关联数据）
- ❌ **禁止**漏 `deletedAt` 字段（统一软删）
- ❌ **禁止**表名复数（`product_items` 错误，应为 `product_item`）
- ❌ **禁止**字段名驼峰在 DB 里（`categoryId` 错误，DB 必须是 `category_id`）
- ❌ **禁止**布尔列加 `is_` 前缀（`isEnabled` 错误，应为 `enabled`）
- ❌ **禁止** `model AdminMenu` 这种 PascalCase 表名（**Prisma 模型 PascalCase，DB 表 snake_case**，`@@map` 桥接）
- ✅ 所有时间字段用 `@db.Timestamptz`（带时区）
- ✅ Decimal 类型必须指定 `(precision, scale)`，不能只写 `@db.Decimal`
- ✅ 任何有排序/筛选需求的字段加 `@@index` 加速分页查询
- ✅ 给业务表加 JSDoc 注释，说明字段用途

## 3. 步骤二：Zod Schema（前后端共享）

> 入参校验一律走 Zod（不写 GraphQL `@Args()` 内联类型），前后端共用一套 schema，**单一数据源**。

### 3.1 新建 `packages/shared/src/schemas/product/product.schema.ts`

```typescript
/**
 * 产品业务 Zod Schema（前后端共享）
 *
 * 使用：
 *   - 后端：resolver 通过 @Args(..., new ZodArgsPipe(Schema)) 校验入参
 *   - 前端：zodToRules → zodToFormRules / zodToPopoverRules 生成表单规则
 */
import { z } from 'zod';

// ============================================================
// 分类
// ============================================================
export const CreateProductCategorySchema = z.object({
    parentId: z.string().uuid().nullable().optional(),
    name: z.string().min(1, '名称不能为空').max(100, '名称不超过 100 字'),
    code: z
        .string()
        .min(2, '编码至少 2 个字符')
        .max(50, '编码不超过 50 个字符')
        .regex(/^[a-z0-9-]+$/, '编码只能包含小写字母、数字、连字符'),
    icon: z.string().max(100).default(''),
    sort: z.number().int().min(0).default(0),
});
export type CreateProductCategoryInput = z.infer<typeof CreateProductCategorySchema>;

export const UpdateProductCategorySchema = CreateProductCategorySchema.partial().extend({
    id: z.string().uuid(),
    enabled: z.boolean().optional(),
});
export type UpdateProductCategoryInput = z.infer<typeof UpdateProductCategorySchema>;

// ============================================================
// 产品
// ============================================================
export const CreateProductItemSchema = z.object({
    categoryId: z.string().uuid('请选择分类'),
    name: z.string().min(1, '名称不能为空').max(200, '名称不超过 200 字'),
    spu: z
        .string()
        .min(2, '产品编码至少 2 个字符')
        .max(50)
        .regex(/^[A-Za-z0-9-]+$/, '编码只能包含字母、数字、连字符'),
    price: z.coerce.number().min(0, '价格不能为负').max(99999999.99),
    stock: z.coerce.number().int().min(0, '库存不能为负').default(0),
    description: z.string().max(5000).default(''),
    coverImage: z.string().max(500).default(''),
    onSale: z.boolean().default(false),
});
export type CreateProductItemInput = z.infer<typeof CreateProductItemSchema>;

export const UpdateProductItemSchema = CreateProductItemSchema.partial().extend({
    id: z.string().uuid(),
    enabled: z.boolean().optional(),
});
export type UpdateProductItemInput = z.infer<typeof UpdateProductItemSchema>;

// ============================================================
// 查询（分页 + 关键字）
// ============================================================
export const ProductItemQuerySchema = z.object({
    keyword: z.string().optional(),
    categoryId: z.string().uuid().optional(),
    onSale: z.boolean().optional(),
    page: z.number().int().min(1).default(1),
    pageSize: z.number().int().min(1).max(100).default(20),
});
export type ProductItemQueryInput = z.infer<typeof ProductItemQuerySchema>;
```

### 4.2 `product-category.type.ts`（ObjectType 纯 GraphQL 输出）

```typescript
 * 输入类型（Create/Update/Query Input）放同目录的 product-category.input.ts，
```

### 4.3 `product-category.service.ts`（业务逻辑）

```typescript
/**
 * 产品分类 Service — 树形 CRUD
 *
 * 关键设计：
 * - 树形列表用单次全表查询 + 内存组装（O(n)），比多次递归查询快 10x
 * - 删除前检查：分类下有子分类 / 有产品 → 拒绝（业务规则）
 * - 所有写操作返回最新实体，方便前端 cache 同步
 */
import { Injectable, NotFoundException, ConflictException } from '@nestjs/common';
import { PrismaService } from '../../common/services/prisma.service.js';
import { uuidv7 } from '../../common/utils/uuid.js';
import type { CreateProductCategoryInput, UpdateProductCategoryInput } from '@packages/shared';

@Injectable()
export class ProductCategoryService {
    constructor(private readonly prisma: PrismaService) {}

    /** 全表 → 树（排除软删） */
    async getTree() {
        const all = await this.prisma.productCategory.findMany({
            where: { deletedAt: null },
            orderBy: [{ sort: 'asc' }, { createdAt: 'asc' }],
        });
        return this.buildTree(all);
    }

    /** 扁平 → 树（O(n)） */
    private buildTree(flat: Array<{ id: string; parentId: string | null }>) {
        const map = new Map<string, any>();
        const roots: any[] = [];
        for (const node of flat) {
            map.set(node.id, { ...node, children: [] });
        }
        for (const node of map.values()) {
            if (node.parentId && map.has(node.parentId)) {
                map.get(node.parentId)!.children.push(node);
            } else {
                roots.push(node);
            }
        }
        return roots;
    }

    async create(input: CreateProductCategoryInput, creatorId: string) {
        // 校验 code 唯一（schema 已加 @unique，这里再 catch 一次给出友好错误）
        const exists = await this.prisma.productCategory.findFirst({
            where: { code: input.code, deletedAt: null },
        });
        if (exists) throw new ConflictException(`分类编码 ${input.code} 已存在`);

        return this.prisma.productCategory.create({
            data: {
                id: uuidv7(),
                parentId: input.parentId ?? null,
                name: input.name,
                code: input.code,
                icon: input.icon ?? '',
                sort: input.sort ?? 0,
                enabled: true,
                createdBy: creatorId,
                updatedBy: creatorId,
            },
        });
    }

    async update(input: UpdateProductCategoryInput, updaterId: string) {
        const cat = await this.prisma.productCategory.findFirst({
            where: { id: input.id, deletedAt: null },
        });
        if (!cat) throw new NotFoundException(`分类 ${input.id} 不存在`);

        // 不能把父级设为自己或自己的后代（防止死循环）
        if (input.parentId) {
            const isDescendant = await this.isDescendant(input.parentId, input.id);
            if (isDescendant) throw new ConflictException('不能将分类的父级设为自己或自己的后代');
        }

        return this.prisma.productCategory.update({
            where: { id: input.id },
            data: {
                ...(input.name !== undefined && { name: input.name }),
                ...(input.parentId !== undefined && { parentId: input.parentId }),
                ...(input.sort !== undefined && { sort: input.sort }),
                ...(input.enabled !== undefined && { enabled: input.enabled }),
                updatedBy: updaterId,
            },
        });
    }

    async softDelete(id: string, operatorId: string) {
        // 检查：是否有子分类
        const childCount = await this.prisma.productCategory.count({
            where: { parentId: id, deletedAt: null },
        });
        if (childCount > 0) throw new ConflictException('存在子分类，无法删除');

        // 检查：是否有关联产品
        const itemCount = await this.prisma.productItem.count({
            where: { categoryId: id, deletedAt: null },
        });
        if (itemCount > 0) throw new ConflictException('该分类下还有产品，无法删除');

        return this.prisma.productCategory.update({
            where: { id },
            data: { deletedAt: new Date(), updatedBy: operatorId },
        });
    }

    private async isDescendant(candidateAncestorId: string, nodeId: string): Promise<boolean> {
        let current = candidateAncestorId;
        const visited = new Set<string>();
        while (current) {
            if (current === nodeId) return true;
            if (visited.has(current)) return false; // 防御性：环
            visited.add(current);
            const parent: { parentId: string | null } | null = await this.prisma.productCategory.findUnique({
                where: { id: current },
                select: { parentId: true },
            });
            current = parent?.parentId ?? '';
        }
        return false;
    }
}
```

### 4.4 `product-category.resolver.ts`（GraphQL 入口）

> **与基座现有模式保持一致**（参考 [`admin-menu.resolver.ts`](./apps/server/src/modules/admin/admin-menu/admin-menu.resolver.ts)）：
> - `@Args(name, { type: () => XxxInputType }, new ZodArgsPipe(Schema))` —— 三个参数一起用：第二个给 GraphQL 生成 schema 用，第三个做业务校验
> - `*.input.ts` 用 `@InputType()` 装饰器类（**不要**塞进 `*.type.ts`，输入与输出严格分文件）
> - 当前账户 ID 通过 `@Context() ctx: GraphQLContext` → `ctx.req.user.accountId` 取

```typescript
/**
 * 产品分类 GraphQL Resolver
 *
 * Query:
 * - productCategoryTree: 获取分类树（一次性拿全）
 * - productCategory(id): 单个详情
 *
 * Mutation:
 * - createProductCategory(input): 新增
 * - updateProductCategory(input): 更新
 * - deleteProductCategory(id): 软删
 *
 * 权限码：
 * - product:category:view   → 所有 Query
 * - product:category:create → createProductCategory
 * - product:category:update → updateProductCategory
 * - product:category:delete → deleteProductCategory
 */
import { UseGuards } from '@nestjs/common';
import { Args, Context, ID, Mutation, Query, Resolver } from '@nestjs/graphql';
import { JwtAuthGuard } from '../../common/guards/jwt-auth.guard.js';
import { AdminPermissionGuard } from '../../common/guards/admin-permission.guard.js';
import { Permission } from '../../common/decorators/permission.decorator.js';
import { RequireAuth } from '../../common/decorators/require-auth.decorator.js';
import { ZodArgsPipe } from '../../common/pipes/zod-args.pipe.js';
import {
    CreateProductCategorySchema,
    UpdateProductCategorySchema,
    UuidSchema,
    type CreateProductCategoryInput,
    type UpdateProductCategoryInput,
} from '@packages/shared';
import type { DataLoaders } from '../../common/dataloader/index.js';
import { ProductCategoryService } from './product-category.service.js';
import { ProductCategory, ProductCategoryTreeNode } from './product-category.type.js';
import { CreateProductCategoryInput as CreateProductCategoryInputType } from './product-category.input.js';
import { UpdateProductCategoryInput as UpdateProductCategoryInputType } from './product-category.input.js';

interface GraphQLContext {
    req: { user: { accountId: string; userType: string } };
    dataloaders?: DataLoaders;
}

@Resolver()
@RequireAuth()
@UseGuards(JwtAuthGuard, AdminPermissionGuard)
export class ProductCategoryResolver {
    constructor(private readonly service: ProductCategoryService) {}

    @Query(() => [ProductCategoryTreeNode], { description: '产品分类树' })
    @Permission('product:category:view')
    async productCategoryTree(): Promise<ProductCategoryTreeNode[]> {
        return this.service.getTree() as Promise<ProductCategoryTreeNode[]>;
    }

    @Query(() => ProductCategory, { description: '单个分类详情', nullable: true })
    @Permission('product:category:view')
    async productCategory(
        @Args('id', { type: () => ID, nullable: false }, new ZodArgsPipe(UuidSchema)) id: string,
    ): Promise<ProductCategory | null> {
        return this.service['prisma'].productCategory.findFirst({
            where: { id, deletedAt: null },
        }) as Promise<ProductCategory | null>;
    }

    @Mutation(() => ProductCategory, { description: '新增产品分类' })
    @Permission('product:category:create')
    async createProductCategory(
        @Context() ctx: GraphQLContext,
        @Args(
            'input',
            { type: () => CreateProductCategoryInputType, nullable: false },
            new ZodArgsPipe(CreateProductCategorySchema),
        )
        input: CreateProductCategoryInput,
    ): Promise<ProductCategory> {
        return this.service.create(input, ctx.req.user.accountId) as Promise<ProductCategory>;
    }

    @Mutation(() => ProductCategory, { description: '更新产品分类' })
    @Permission('product:category:update')
    async updateProductCategory(
        @Context() ctx: GraphQLContext,
        @Args(
            'input',
            { type: () => UpdateProductCategoryInputType, nullable: false },
            new ZodArgsPipe(UpdateProductCategorySchema),
        )
        input: UpdateProductCategoryInput,
    ): Promise<ProductCategory> {
        return this.service.update(input, ctx.req.user.accountId) as Promise<ProductCategory>;
    }

    @Mutation(() => Boolean, { description: '软删产品分类' })
    @Permission('product:category:delete')
    async deleteProductCategory(
        @Context() ctx: GraphQLContext,
        @Args('id', { type: () => ID, nullable: false }, new ZodArgsPipe(UuidSchema)) id: string,
    ): Promise<boolean> {
        await this.service.softDelete(id, ctx.req.user.accountId);
        return true;
    }
}
```

```typescript
/**
 * 产品分类 API（GraphQL）
 *
 * 权限码：
 *   - product:category:view   → productCategoryTree / productCategory
 *   - product:category:create → createProductCategory
 *   - product:category:update → updateProductCategory
 *   - product:category:delete → deleteProductCategory
 */
import { gqlQuery } from '@/shared/request/graphql-client';

// ============================================================
// 类型（与后端 ObjectType 字段一一对应）
// ============================================================
export interface ProductCategoryNode {
    id: string;
    parentId: string | null;
    name: string;
    code: string;
    icon: string;
    sort: number;
    enabled: boolean;
    children: ProductCategoryNode[];
}

// ============================================================
// Query
// ============================================================
export async function getProductCategoryTree(): Promise<ProductCategoryNode[]> {
    const data = await gqlQuery<{ productCategoryTree: ProductCategoryNode[] }>(`
        query ProductCategoryTree {
            productCategoryTree {
                id parentId name code icon sort enabled
                children {
                    id parentId name code icon sort enabled
                    children { id parentId name code icon sort enabled }
                }
            }
        }
    `);
    return data.productCategoryTree;
}

export async function getProductCategory(id: string): Promise<ProductCategoryNode | null> {
    const data = await gqlQuery<{ productCategory: ProductCategoryNode | null }>(
        `query ProductCategory($id: ID!) { productCategory(id: $id) { id parentId name code icon sort enabled } }`,
        { variables: { id } },
    );
    return data.productCategory;
}

// ============================================================
// Mutation（入参结构由后端 Zod 校验）
// ============================================================
export async function createProductCategory(input: {
    parentId?: string | null;
    name: string;
    code: string;
    icon?: string;
    sort?: number;
}): Promise<ProductCategoryNode> {
    const data = await gqlQuery<{ createProductCategory: ProductCategoryNode }>(
        `mutation Create($input: CreateProductCategoryInput!) {
            createProductCategory(input: $input) { id parentId name code icon sort enabled }
        }`,
        { variables: { input } },
    );
    return data.createProductCategory;
}

export async function updateProductCategory(input: {
    id: string;
    name?: string;
    parentId?: string | null;
    sort?: number;
    enabled?: boolean;
}): Promise<ProductCategoryNode> {
    const data = await gqlQuery<{ updateProductCategory: ProductCategoryNode }>(
        `mutation Update($input: UpdateProductCategoryInput!) {
            updateProductCategory(input: $input) { id parentId name code icon sort enabled }
        }`,
        { variables: { input } },
    );
    return data.updateProductCategory;
}

export async function deleteProductCategory(id: string): Promise<boolean> {
    const data = await gqlQuery<{ deleteProductCategory: boolean }>(
        `mutation Delete($id: ID!) { deleteProductCategory(id: $id) }`,
        { variables: { id } },
    );
    return data.deleteProductCategory;
}
```

## 13. 常见坑（Fork 后必读）

1. **改了菜单但用户看不到** → 没调 `invalidateCaches()`，旧缓存 30 分钟内生效。手动 `pnpm exec tsx prisma/seed.ts` 一次，或删 `mono:auth:*` Redis 键
2. **GraphQL 字段返回 null 但类型是非 null** → 检查后端 `ObjectType` 的 `nullable: true/false`，与 Prisma schema 字段的可空性对齐
3. **前端 `meta.permissions` 不生效** → 检查 seed 里 `permissionCode` 是否填了（路由级权限）
4. **按钮看不见但接口能调通** → 改后端 `AdminPermissionGuard`（双重防御，参考 [开发文档/权限控制.md](./开发文档/权限控制.md)）
5. **改了 Prisma schema 但 Prisma Client 没更新** → 跑 `pnpm db:generate`
6. **测试覆盖率下降导致 CI 挂** → 新增文件必须配测试，**没有例外**
7. **临时 SQL 写到 `/tmp/`** → 一律放 `.tmp/`，任务结束 `rm -rf .tmp/*` 清空
8. **菜单排序乱** → 检查 seed 里 `sort` 字段，**sort 升序**，整型
9. **前端动态路由没注册** → 检查 `componentMap` 是否加了对应 key
10. **新增 RESTful controller** → **违规**！业务模块 100% GraphQL，REST 仅用于 BFF 层（认证、上传、健康检查）
11. **写 GraphQL InputType 只用 Zod schema 不用 `@InputType()` 装饰器** → **违规**！GraphQL schema 生成需要 `@InputType()` 元数据，必须在 `*.input.ts` 配 `@InputType()` 类 + Zod schema，二者用 `@Args(name, { type: () => XxxInputType, nullable: false }, new ZodArgsPipe(Schema))` 三参数绑定
12. **手搓 `<div class="p-6 space-y-4">` 布局** → **违规**！用 `<n-card title>` + `<template #header-extra>`（[布局与UI规范 §6.1](./开发文档/布局与UI规范.md#六页面结构)）
