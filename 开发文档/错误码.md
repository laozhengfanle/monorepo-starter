# 错误码

> REST 和 GraphQL 共用同一套业务错误码。前端无论走哪种协议，`code` 含义一致。
>
> **权威源**：后端 `apps/server/src/common/errors/error-codes.ts` 是错误码数值的唯一定义源，前端 `packages/shared/src/errors/error-codes.ts` 同步维护（两者通过 `packages/shared/src/errors/__tests__/error-codes.spec.ts` 的一致性测试保证对齐）。本文档与代码保持一致，新增错误码时先改代码再同步本文档。

## 码段总览

> 快速索引：5 位错误码的前 2 位代表模块，便于一眼定位错误所属子系统。

| 段位     | 模块               | 详细定义                                                |
| -------- | ------------------ | ------------------------------------------------------- |
| `10xxx`  | 通用 / 参数 / 系统 | 详见下方"通用"章节                                      |
| `11xxx`  | 账户操作           | 详见下方"账户操作"章节                                  |
| `12xxx`  | 系统配置 / 文件    | 详见下方"系统配置/文件"章节                             |
| `20xxx`  | 认证 / 鉴权        | 详见下方"认证"章节                                      |
| `21xxx`  | 用户管理           | 详见下方"用户管理"章节                                  |
| `22xxx`  | 角色 / 权限        | 详见下方"权限"章节                                      |
| `30xxx`  | 短信服务           | 详见 [短信服务方案.md](./短信服务方案.md)               |
| `301xx`  | 邮件服务           | 详见 [邮件服务方案.md](./邮件服务方案.md)               |
| `31xxx+` | 业务模块           | 按业务子域分配（产品/订单/课程等）                      |
| `40xxx`  | OAuth 第三方登录   | 详见 [oauth第三方登录方案.md](./oauth第三方登录方案.md) |

> 引用关系：本文档是错误码体系的**总入口与权威源**，各模块的详细错误码子集在对应模块文档中列出。

## 规范

```text
规则：5 位数字，前 2 位为模块编号，后 3 位为具体错误
```

> **原则**：HTTP 状态码只表示传输层结果（200/400/401/403/500），业务结果一律用 `code` 字段区分。

### 错误码格式约定

| 协议    | code 字段类型 | 示例      | 说明                                                     |
| ------- | ------------- | --------- | -------------------------------------------------------- |
| REST    | `number`      | `10001`   | JSON 响应中 code 为数字类型                              |
| GraphQL | `string`      | `"10001"` | GraphQL extensions.code 为字符串类型（GraphQL 规范要求） |

两端使用同一套 5 位数字错误码，仅传输格式不同。GraphQL ExceptionFilter 在转换时自动 `String(errorCode)`。

## REST 响应体

```typescript
// 成功
{
  code: 0,
  message: "success",
  data: T
}

// 分页
{
  code: 0,
  message: "success",
  data: { items: T[], total: number, page: number, pageSize: number }
}

// 错误
{
  code: 10001,
  message: "参数验证失败",
  data: null
}
```

## GraphQL 错误

```json
{
    "errors": [
        {
            "message": "参数验证失败",
            "extensions": {
                "code": "10001",
                "fields": [{ "field": "username", "message": "不能为空" }]
            }
        }
    ]
}
```

GraphQL 的 `code` 与 REST 的 `code` 含义一致，由 `GraphQLExceptionFilter` 统一映射。

> **类型差异说明**：REST 响应体中 `code` 为**数字**（`code: 10001`），GraphQL 错误中 `code` 为**字符串**（`"code": "10001"`）。这是因为 GraphQL 的 `extensions` 必须是可序列化的 JSON 值，且 `GraphQLExceptionFilter` 内部通过 `String(response.code)` 转换。前端解析时需注意类型差异，建议统一按字符串处理（`String(code)` 或 `Number(code)`）。
