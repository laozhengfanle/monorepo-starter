# 短信服务方案

> C 端核心依赖：手机号 + 验证码是 C 端主要登录方式。本文定义服务商选型、API 封装、验证码生命周期和降级策略。
> 设计原则：**服务商可替换、后台可配置**——管理后台修改 `system_config` 表即可切换服务商/密钥/模板，无需改代码、无需重启服务。

---

## 一、服务商选型

### 候选对比

| 维度         | 阿里云短信    | 腾讯云短信    |
| ------------ | ------------- | ------------- |
| 国内到达率   | ≥ 99%         | ≥ 99%         |
| 验证码单价   | ¥0.045/条     | ¥0.045/条     |
| 国际短信     | 支持（贵）    | 支持（贵）    |
| SDK 质量     | ⭐⭐⭐        | ⭐⭐⭐        |
| 模板审核速度 | 1-2h          | 1-2h          |
| 签名管理     | 控制台 + API  | 控制台 + API  |
| 回调支持     | 发送状态回调  | 发送状态回调  |
| 免费额度     | 新用户 200 条 | 新用户 100 条 |

### 推荐方案

**默认使用阿里云短信**，理由：

1. SDK 成熟，`@alicloud/dysmsapi20170525` 维护活跃
2. 国内到达率稳定，模板审核快
3. 与项目已有阿里云生态（OSS 等）统一管理

> fork 后如需切换，在管理后台修改 `system_config` 表中的 `sms.provider` 配置即可，业务代码无需改动。

### system_config 配置结构

短信相关配置存储在 `system_config` 表中，管理后台可动态修改：

| key                    | value 结构  | 说明                                             |
| ---------------------- | ----------- | ------------------------------------------------ |
| `sms.provider`         | 见下方 JSON | 短信服务商主配置（driver、签名、模板、限流参数） |
| `sms.provider.aliyun`  | 见下方 JSON | 阿里云密钥（敏感，API 返回时脱敏）               |
| `sms.provider.tencent` | 见下方 JSON | 腾讯云密钥（敏感，API 返回时脱敏）               |

```typescript
/** sms.provider 配置结构 */
interface SmsProviderConfig {
    /** 当前使用的服务商：aliyun | tencent | ... */
    driver: string;
    /** 短信签名（需在服务商控制台提前审核通过） */
    signName: string;
    /** 各用途对应的模板 Code */
    templates: {
        login: string;
        register: string;
        reset_password: string;
        bind_phone: string;
    };
    /** 频率限制参数 */
    limits: {
        interval: number; // 发送间隔（秒），默认 60
        daily: number; // 每日发送上限，默认 10
        ipHourly: number; // IP 每小时上限，默认 20
        codeTtl: number; // 验证码有效期（秒），默认 300
        maxAttempts: number; // 验证码最大尝试次数，默认 5
    };
    /** 降级配置 */
    fallback: {
        turnstileEnabled: boolean; // 短信失败时是否降级为图形验证码（Turnstile）
    };
}

/** sms.provider.aliyun 密钥配置 */
interface AliyunSmsCredential {
    accessKeyId: string;
    accessKeySecret: string;
}

/** sms.provider.tencent 密钥配置 */
interface TencentSmsCredential {
    secretId: string;
    secretKey: string;
    sdkAppId: string;
}
```

### SmsService 业务层

```typescript
@Injectable()
export class SmsService {
    constructor(
        @Inject('SMS_PROVIDER') private readonly provider: SmsProvider,
        private readonly sysConfig: SystemConfigService,
        private readonly cache: ICacheService,
    ) {}

    /**
     * 发送验证码
     * 1. 从 system_config 读取限流参数
     * 2. 检查发送频率限制
     * 3. 生成 6 位数字验证码
     * 4. 存入 Redis（TTL 由 system_config 配置）
     * 5. 从 system_config 读取签名和模板，调用 SmsProvider 发送
     * 6. 可选：写入审计表
     */
    async sendVerificationCode(phone: string, purpose: SmsPurpose, ip: string): Promise<void> {
        // 从 system_config 读取配置（优先走 Redis 缓存，miss 时查 DB）
        const config = await this.sysConfig.getConfig<SmsProviderConfig>('sms.provider');
        const { limits, signName, templates } = config;

        // 频率限制检查
        await this.checkRateLimit(phone, ip, limits);

        // 生成验证码
        const code = this.generateCode();

        // 存入 Redis（TTL 由配置决定）
        const redisKey = `mono:verify:sms:${phone}`;
        await this.cache.setex(
            redisKey,
            limits.codeTtl,
            JSON.stringify({
                code,
                purpose,
                attempts: 0,
            }),
        );

        // 发送短信
        const result = await this.provider.send({
            phone,
            templateCode: templates[purpose],
            templateParams: { code },
            signName,
        });

        if (!result.success) {
            // 发送失败，清除 Redis 中的验证码
            await this.cache.del(redisKey);
            throw new BusinessException(ErrorCode.SMS_SEND_FAILED, result.errorMessage);
        }

        // 设置发送间隔标记
        await this.cache.setex(`mono:verify:sms:interval:${phone}`, limits.interval, '1');

        // 审计记录（可选，见第四节）
        await this.recordAudit(phone, purpose, ip, result.requestId);
    }

    /**
     * 校验验证码
     * 返回 true 表示验证通过，同时删除验证码（一次性使用）
     */
    async verifyCode(phone: string, code: string, purpose: SmsPurpose): Promise<boolean> {
        const config = await this.sysConfig.getConfig<SmsProviderConfig>('sms.provider');
        const { maxAttempts } = config.limits;

        const redisKey = `mono:verify:sms:${phone}`;
        const stored = await this.cache.get(redisKey);

        if (!stored) {
            throw new BusinessException(ErrorCode.SMS_CODE_EXPIRED);
        }

        const data = JSON.parse(stored);

        // 用途不匹配（如用"重置密码"的验证码去"登录"）
        if (data.purpose !== purpose) {
            throw new BusinessException(ErrorCode.SMS_CODE_PURPOSE_MISMATCH);
        }

        // 错误次数检查
        if (data.attempts >= maxAttempts) {
            await this.cache.del(redisKey);
            throw new BusinessException(ErrorCode.SMS_CODE_MAX_ATTEMPTS);
        }

        if (data.code !== code) {
            data.attempts += 1;
            await this.cache.setex(redisKey, await this.cache.ttl(redisKey), JSON.stringify(data));
            throw new BusinessException(ErrorCode.SMS_CODE_INVALID);
        }

        // 验证通过，删除验证码
        await this.cache.del(redisKey);
        return true;
    }

    /** 频率限制检查（参数从 system_config 读取） */
    private async checkRateLimit(phone: string, ip: string, limits: SmsProviderConfig['limits']): Promise<void> {
        // 发送间隔检查
        const intervalKey = `mono:verify:sms:interval:${phone}`;
        if (await this.cache.exists(intervalKey)) {
            throw new BusinessException(ErrorCode.SMS_SEND_TOO_FREQUENT);
        }

        // 同一手机号每日上限
        const dailyKey = `mono:verify:sms:daily:${phone}:${this.getToday()}`;
        const dailyCount = await this.cache.incr(dailyKey);
        if (dailyCount === 1) {
            // 首次计数，设置 TTL（incr 后无法原子性设 TTL，此处用 setex 重写）
            await this.cache.setex(dailyKey, 86400, 1);
        }
        if (dailyCount > limits.daily) {
            throw new BusinessException(ErrorCode.SMS_DAILY_LIMIT_EXCEEDED);
        }

        // 同一 IP 每小时上限（防刷）
        const ipHourlyKey = `mono:verify:sms:ip:${ip}:${this.getCurrentHour()}`;
        const ipCount = await this.cache.incr(ipHourlyKey);
        if (ipCount === 1) {
            await this.cache.setex(ipHourlyKey, 3600, 1);
        }
        if (ipCount > limits.ipHourly) {
            throw new BusinessException(ErrorCode.SMS_IP_LIMIT_EXCEEDED);
        }
    }

    /** 生成 6 位数字验证码 */
    private generateCode(): string {
        return crypto.randomInt(100000, 999999).toString();
    }

    private getToday(): string {
        return new Date().toISOString().slice(0, 10);
    }

    private getCurrentHour(): string {
        return new Date().toISOString().slice(0, 13);
    }
}

type SmsPurpose = 'login' | 'register' | 'reset_password' | 'bind_phone';
```

### SmsModule 动态模块

```typescript
@Module({})
export class SmsModule {
    /**
     * 从 system_config 表读取 sms.provider.driver，动态注入对应实现
     * 管理后台修改 driver 后，清除 Redis 缓存即可即时切换
     */
    static forRoot(): DynamicModule {
        return {
            module: SmsModule,
            providers: [
                {
                    provide: 'SMS_PROVIDER',
                    useFactory: async (sysConfig: SystemConfigService) => {
                        const config = await sysConfig.getConfig<SmsProviderConfig>('sms.provider');
                        const driver = config.driver;

                        // 根据 driver 从 system_config 读取对应密钥，创建 Provider 实例
                        const providerMap: Record<
                            string,
                            (config: SmsProviderConfig, sysConfig: SystemConfigService) => Promise<SmsProvider>
                        > = {
                            aliyun: async (cfg, sc) => {
                                const cred = await sc.getConfig<AliyunSmsCredential>('sms.provider.aliyun');
                                return new AliyunSmsProvider(cred.accessKeyId, cred.accessKeySecret);
                            },
                            tencent: async (cfg, sc) => {
                                const cred = await sc.getConfig<TencentSmsCredential>('sms.provider.tencent');
                                return new TencentSmsProvider(cred.secretId, cred.secretKey, cred.sdkAppId);
                            },
                        };

                        const factory = providerMap[driver];
                        if (!factory) {
                            throw new Error(
                                `Unsupported SMS provider: ${driver}. Available: ${Object.keys(providerMap).join(', ')}`,
                            );
                        }

                        return factory(config, sysConfig);
                    },
                    inject: [SystemConfigService],
                },
                SmsService,
            ],
            exports: [SmsService],
        };
    }
}
```

### SystemConfigService 系统配置服务

```typescript
/**
 * 系统配置服务
 * 读取 system_config 表的配置，优先走 Redis 缓存
 * 管理后台修改配置后，自动清除 Redis 缓存，确保即时生效
 */
@Injectable()
export class SystemConfigService {
    private readonly CACHE_PREFIX = 'mono:data:system_config:';
    private readonly CACHE_TTL = 300; // 5 分钟缓存

    constructor(
        private readonly prisma: PrismaService,
        private readonly cache: ICacheService,
    ) {}

    /**
     * 获取配置
     * 1. 先查 Redis 缓存
     * 2. miss 时查 DB，写入 Redis
     */
    async getConfig<T>(key: string): Promise<T> {
        const cacheKey = `${this.CACHE_PREFIX}${key}`;

        // 1. 查 Redis
        const cached = await this.cache.get(cacheKey);
        if (cached) {
            return JSON.parse(cached);
        }

        // 2. 查 DB
        const row = await this.prisma.system_config.findUnique({
            where: { key },
        });

        if (!row) {
            throw new Error(`SystemConfig not found: ${key}`);
        }

        // 3. 写入 Redis
        await this.cache.setex(cacheKey, this.CACHE_TTL, JSON.stringify(row.value));

        return row.value as T;
    }

    /**
     * 更新配置（管理后台调用）
     * 1. 更新 DB
     * 2. 删除 Redis 缓存（下次读取时自动重建）
     */
    async updateConfig(key: string, value: unknown, updatedBy: string): Promise<void> {
        await this.prisma.system_config.upsert({
            where: { key },
            update: { value, updated_by: updatedBy, updated_at: new Date() },
            create: { id: generateUuidV7(), key, value, updated_by: updatedBy },
        });

        // 清除缓存，确保即时生效
        await this.cache.del(`${this.CACHE_PREFIX}${key}`);
    }

    /**
     * 获取配置（脱敏版，管理后台展示用）
     * 敏感字段（密钥、密码）只显示后 4 位
     */
    async getConfigMasked(key: string): Promise<unknown> {
        const value = await this.getConfig<Record<string, unknown>>(key);
        return this.maskSensitiveFields(value);
    }

    /** 递归脱敏：对包含 key/secret/password/token 的字段做掩码处理 */
    private maskSensitiveFields(obj: Record<string, unknown>): Record<string, unknown> {
        const sensitivePatterns = [/key/i, /secret/i, /password/i, /token/i];
        const result: Record<string, unknown> = {};

        for (const [k, v] of Object.entries(obj)) {
            if (typeof v === 'string' && sensitivePatterns.some((p) => p.test(k))) {
                // 敏感字段只显示后 4 位
                result[k] = v.length > 4 ? `****${v.slice(-4)}` : '****';
            } else if (typeof v === 'object' && v !== null && !Array.isArray(v)) {
                result[k] = this.maskSensitiveFields(v as Record<string, unknown>);
            } else {
                result[k] = v;
            }
        }

        return result;
    }
}
```

## 三、验证码生命周期

```text
发送请求
  │
  ├─ 从 system_config 读取限流参数
  │
  ├─❌ 频率限制检查失败 → 拒绝发送
  │
  ├─✅ 通过
  │   │
  │   ├─ 生成 6 位验证码
  │   ├─ 存入 Redis（TTL 由 system_config 配置，默认 5min）
  │   │   key:   mono:verify:sms:{phone}
  │   │   value: { code, purpose, attempts: 0 }
  │   │
  │   ├─ 从 system_config 读取签名和模板
  │   ├─ 调用 SmsProvider.send()
  │   │   ├─❌ 发送失败 → 删除 Redis key → 返回错误
  │   │   └─✅ 发送成功 → 设置间隔标记
  │   │
  │   └─ 审计记录（可选）
  │
验证请求
  │
  ├─ Redis key 不存在 → 验证码过期
  ├─ purpose 不匹配 → 用途错误
  ├─ attempts ≥ maxAttempts → 超过最大尝试次数，删除 key
  ├─ code 不匹配 → attempts++，返回错误
  └─✅ code 匹配 → 删除 key（一次性使用）→ 验证通过
```

### Redis Key 汇总

| Key                                    | 类型          | TTL                  | 说明                     |
| -------------------------------------- | ------------- | -------------------- | ------------------------ |
| `mono:data:system_config:{key}`        | STRING (JSON) | 5 min                | system_config 缓存       |
| `mono:verify:sms:{phone}`              | STRING (JSON) | 可配置（默认 5 min） | 验证码 + 用途 + 尝试次数 |
| `mono:verify:sms:interval:{phone}`     | STRING        | 可配置（默认 60 s）  | 发送间隔锁               |
| `mono:verify:sms:daily:{phone}:{date}` | INT           | 24 h                 | 同一手机号每日发送计数   |
| `mono:verify:sms:ip:{ip}:{hour}`       | INT           | 1 h                  | 同一 IP 每小时发送计数   |

> 与 [缓存设计.md](./缓存设计.md) 的 key 命名规范一致：`mono:{domain}:{entity}:{identifier}`。

## 四、审计追溯

### 方案选择

| 方案                             | 优点                                   | 缺点                                         | 推荐 |
| -------------------------------- | -------------------------------------- | -------------------------------------------- | :--: |
| **A: Redis Only**                | 简单，无需额外表                       | 无法历史回溯，TTL 过期即丢失                 |  ❌  |
| **B: 写入 verification_code 表** | 完整审计，可查"某手机号某天发了多少次" | 多一次 DB 写入                               |  ✅  |
| **C: 写入 audit_log 表**         | 复用现有审计体系                       | audit_log 表膨胀快，验证码审计与安全审计耦合 |  ❌  |

**推荐方案 B**：启用 [数据库设计.md](./数据库设计.md#10-verification_code-验证码审计表) 中预留的 `verification_code` 表。

### 审计记录逻辑

```typescript
/**
 * 发送成功后写入审计表
 * 此写入不影响主流程——写失败只记日志，不抛异常
 */
private async recordAudit(phone: string, purpose: SmsPurpose, ip: string, requestId: string): Promise<void> {
  try {
    await this.prisma.verification_code.create({
      data: {
        id: generateUuidV7(),
        identifier: phone,
        code: '******', // 审计表不存明文验证码，只记录"发送了"
        purpose,
        ip,
        expires_at: new Date(Date.now() + 5 * 60 * 1000),
      },
    });
  } catch (error) {
    // 审计写入失败不影响主流程
    this.logger.warn(`Failed to write SMS audit record: ${error.message}`);
  }
}
```

> ⚠️ 审计表 **不存明文验证码**（`code: '******'`），明文只存 Redis（5 分钟 TTL）。这是安全合规要求——数据库被拖库后无法获取有效验证码。

## 五、降级策略

### 场景与应对

| 场景             | 影响                | 应对策略                                                                    |
| ---------------- | ------------------- | --------------------------------------------------------------------------- |
| 服务商 API 超时  | 验证码发不出去      | 1. 超时 3s 自动重试 1 次 2. 记录失败日志 3. 返回友好错误提示                |
| 服务商整体故障   | 短信通道不可用      | 1. 管理后台切换到备用服务商 2. 降级为图形验证码（Turnstile）3. 告警通知运维 |
| Redis 不可用     | 验证码无法存储/校验 | 1. 降级为数据库 verification_code 表做临时存储 2. 性能下降但功能可用        |
| 手机号收不到短信 | 用户无法登录        | 1. 提供语音验证码备选（阿里云支持） 2. 引导用户使用微信登录                 |

### 降级代码示例

```typescript
@Injectable()
export class SmsService {
    /**
     * 发送验证码（带降级）
     * 短信失败时，如配置了 Turnstile 降级，返回提示引导用户走图形验证
     */
    async sendVerificationCode(phone: string, purpose: SmsPurpose, ip: string): Promise<void> {
        try {
            await this.sendViaSms(phone, purpose, ip);
        } catch (smsError) {
            this.logger.warn(`SMS send failed for ${phone}: ${smsError.message}`);

            // 从 system_config 读取降级配置
            const config = await this.sysConfig.getConfig<SmsProviderConfig>('sms.provider');
            if (config.fallback.turnstileEnabled) {
                throw new BusinessException(ErrorCode.SMS_SEND_FAILED_BUT_TURNSTILE_FALLBACK);
            }

            throw new BusinessException(ErrorCode.SMS_SEND_FAILED);
        }
    }
}
```

## 六、管理后台配置

### 配置页面

## 九、验证码服务统一抽象层

> 短信验证码和邮件验证码共享相同的生命周期（生成 → 存储 → 校验 → 失效），
> 只有"发送"环节不同（短信走 SmsProvider，邮件走 EmailProvider）。
> 因此抽象出 `VerificationCodeService` 统一处理验证码逻辑，短信/邮件只负责投递。

### 架构

```text
VerificationCodeService（统一验证码服务）
  │
  ├─ 生成验证码（6 位数字 / 随机字母数字）
  ├─ 存储到 Redis（TTL、purpose、attempts）
  ├─ 频率限制检查（interval / daily / ipHourly）
  ├─ 校验验证码（匹配 → 删除，不匹配 → attempts++）
  ├─ 审计记录（写入 verification_code 表）
  │
  ├─→ SmsProvider.send()    # 短信渠道投递
  └─→ EmailProvider.send()  # 邮件渠道投递
```

### 接口定义

```typescript
/**
 * 验证码渠道提供者接口
 * 短信和邮件各自实现此接口，只负责"投递"验证码
 */
export interface VerificationChannelProvider {
    /** 渠道标识：sms | email */
    readonly channel: string;

    /**
     * 投递验证码
     * @param identifier 接收标识（手机号 E.164 或邮箱地址）
     * @param code 验证码明文
     * @param purpose 用途
     * @param templateConfig 模板配置（从 system_config 读取）
     */
    deliver(params: VerificationDeliverParams): Promise<VerificationDeliverResult>;
}

export interface VerificationDeliverParams {
    /** 接收标识（手机号或邮箱） */
    identifier: string;
    /** 验证码明文 */
    code: string;
    /** 用途：login / register / reset_password / bind_phone / verify_email */
    purpose: VerificationPurpose;
    /** 模板配置（各渠道自定义） */
    templateConfig: Record<string, unknown>;
}

export interface VerificationDeliverResult {
    /** 是否投递成功 */
    success: boolean;
    /** 服务商回执 ID */
    requestId: string;
    /** 失败时的错误码 */
    errorCode?: string;
    /** 失败时的错误信息 */
    errorMessage?: string;
}

type VerificationPurpose = 'login' | 'register' | 'reset_password' | 'bind_phone' | 'verify_email';
```

### VerificationCodeService 实现

```typescript
/**
 * 统一验证码服务
 * 处理验证码的生成、存储、校验、限流、审计
 * 短信和邮件共用此服务，只通过不同的 ChannelProvider 投递
 */
@Injectable()
export class VerificationCodeService {
    private readonly logger = new Logger(VerificationCodeService.name);

    /** 渠道提供者注册表：channel → provider 实例 */
    private readonly channelProviders = new Map<string, VerificationChannelProvider>();

    constructor(
        private readonly sysConfig: SystemConfigService,
        private readonly cache: ICacheService,
        private readonly prisma: PrismaService,
    ) {}

    /**
     * 注册渠道提供者（在模块初始化时调用）
     * SmsModule 和 EmailModule 各自注册自己的 provider
     */
    registerProvider(provider: VerificationChannelProvider): void {
        this.channelProviders.set(provider.channel, provider);
    }

    /**
     * 发送验证码（统一入口）
     * 1. 根据 channel 获取对应的 provider
     * 2. 从 system_config 读取限流参数
     * 3. 频率限制检查
     * 4. 生成验证码
     * 5. 存入 Redis
     * 6. 调用 provider 投递
     * 7. 审计记录
     */
    async sendCode(
        identifier: string,
        purpose: VerificationPurpose,
        channel: 'sms' | 'email',
        ip: string,
    ): Promise<void> {
        // 1. 获取渠道提供者
        const provider = this.channelProviders.get(channel);
        if (!provider) {
            throw new BusinessException(ErrorCode.VERIFICATION_CHANNEL_NOT_SUPPORTED);
        }

        // 2. 从 system_config 读取配置
        const configKey = channel === 'sms' ? 'sms.provider' : 'email.provider';
        const config = await this.sysConfig.getConfig<VerificationConfig>(configKey);

        // 3. 频率限制检查
        await this.checkRateLimit(identifier, ip, channel, config.limits);

        // 4. 生成验证码
        const code = this.generateCode();

        // 5. 存入 Redis
        const redisKey = `mono:verify:${channel}:${identifier}`;
        await this.cache.setex(
            redisKey,
            config.limits.codeTtl,
            JSON.stringify({
                code,
                purpose,
                channel,
                attempts: 0,
            }),
        );

        // 6. 调用 provider 投递
        const result = await provider.deliver({
            identifier,
            code,
            purpose,
            templateConfig: config.templates || {},
        });

        if (!result.success) {
            // 投递失败，清除 Redis 中的验证码
            await this.cache.del(redisKey);
            throw new BusinessException(ErrorCode.VERIFICATION_SEND_FAILED, result.errorMessage);
        }

        // 设置发送间隔标记
        await this.cache.setex(`mono:verify:${channel}:interval:${identifier}`, config.limits.interval, '1');

        // 7. 审计记录
        await this.recordAudit(identifier, purpose, channel, ip, result.requestId);
    }

    /**
     * 校验验证码（统一入口）
     * 短信和邮件验证码共用同一套校验逻辑
     */
    async verifyCode(
        identifier: string,
        code: string,
        purpose: VerificationPurpose,
        channel: 'sms' | 'email',
    ): Promise<boolean> {
        const configKey = channel === 'sms' ? 'sms.provider' : 'email.provider';
        const config = await this.sysConfig.getConfig<VerificationConfig>(configKey);
        const { maxAttempts } = config.limits;

        const redisKey = `mono:verify:${channel}:${identifier}`;
        const stored = await this.cache.get(redisKey);

        if (!stored) {
            throw new BusinessException(ErrorCode.VERIFICATION_CODE_EXPIRED);
        }

        const data = JSON.parse(stored);

        // 用途不匹配
        if (data.purpose !== purpose) {
            throw new BusinessException(ErrorCode.VERIFICATION_CODE_PURPOSE_MISMATCH);
        }

        // 渠道不匹配（如用短信验证码去邮件端点校验）
        if (data.channel !== channel) {
            throw new BusinessException(ErrorCode.VERIFICATION_CODE_CHANNEL_MISMATCH);
        }

        // 错误次数检查
        if (data.attempts >= maxAttempts) {
            await this.cache.del(redisKey);
            throw new BusinessException(ErrorCode.VERIFICATION_CODE_MAX_ATTEMPTS);
        }

        if (data.code !== code) {
            data.attempts += 1;
            await this.cache.setex(redisKey, await this.cache.ttl(redisKey), JSON.stringify(data));
            throw new BusinessException(ErrorCode.VERIFICATION_CODE_INVALID);
        }

        // 验证通过，删除验证码
        await this.cache.del(redisKey);

        // 更新审计记录状态
        await this.updateAuditStatus(identifier, purpose, channel, 'verified');

        return true;
    }

    /** 频率限制检查（短信和邮件共用） */
    private async checkRateLimit(
        identifier: string,
        ip: string,
        channel: string,
        limits: VerificationLimits,
    ): Promise<void> {
        // 发送间隔检查
        const intervalKey = `mono:verify:${channel}:interval:${identifier}`;
        if (await this.cache.exists(intervalKey)) {
            throw new BusinessException(ErrorCode.VERIFICATION_SEND_TOO_FREQUENT);
        }

        // 同一标识每日上限
        const dailyKey = `mono:verify:${channel}:daily:${identifier}:${this.getToday()}`;
        const dailyCount = await this.cache.incr(dailyKey);
        if (dailyCount === 1) {
            await this.cache.setex(dailyKey, 86400, 1);
        }
        if (dailyCount > limits.daily) {
            throw new BusinessException(ErrorCode.VERIFICATION_DAILY_LIMIT_EXCEEDED);
        }

        // 同一 IP 每小时上限（防刷）
        const ipHourlyKey = `mono:verify:${channel}:ip:${ip}:${this.getCurrentHour()}`;
        const ipCount = await this.cache.incr(ipHourlyKey);
        if (ipCount === 1) {
            await this.cache.setex(ipHourlyKey, 3600, 1);
        }
        if (ipCount > limits.ipHourly) {
            throw new BusinessException(ErrorCode.VERIFICATION_IP_LIMIT_EXCEEDED);
        }
    }

    /** 生成 6 位数字验证码 */
    private generateCode(): string {
        return crypto.randomInt(100000, 999999).toString();
    }

    /** 审计记录（写入 verification_code 表） */
    private async recordAudit(
        identifier: string,
        purpose: VerificationPurpose,
        channel: string,
        ip: string,
        requestId: string,
    ): Promise<void> {
        try {
            await this.prisma.verification_code.create({
                data: {
                    id: generateUuidV7(),
                    identifier,
                    code: '******', // 审计表不存明文
                    purpose,
                    channel,
                    ip,
                    status: 'sent',
                    expires_at: new Date(Date.now() + 5 * 60 * 1000),
                },
            });
        } catch (error) {
            this.logger.warn(`Failed to write verification audit record: ${error.message}`);
        }
    }

    /** 更新审计记录状态（验证通过时） */
    private async updateAuditStatus(
        identifier: string,
        purpose: string,
        channel: string,
        status: string,
    ): Promise<void> {
        try {
            // 找到最近一条 sent 状态的记录，更新为 verified
            const record = await this.prisma.verification_code.findFirst({
                where: { identifier, purpose, channel, status: 'sent' },
                orderBy: { created_at: 'desc' },
            });
            if (record) {
                await this.prisma.verification_code.update({
                    where: { id: record.id },
                    data: { status, verified_at: new Date() },
                });
            }
        } catch (error) {
            this.logger.warn(`Failed to update verification audit status: ${error.message}`);
        }
    }

    private getToday(): string {
        return new Date().toISOString().slice(0, 10);
    }

    private getCurrentHour(): string {
        return new Date().toISOString().slice(0, 13);
    }
}

/** 验证码配置结构（短信和邮件共用） */
interface VerificationConfig {
    driver: string;
    limits: VerificationLimits;
    templates: Record<string, unknown>;
}

interface VerificationLimits {
    interval: number; // 发送间隔（秒）
    daily: number; // 每日发送上限
    ipHourly: number; // IP 每小时上限
    codeTtl: number; // 验证码有效期（秒）
    maxAttempts: number; // 最大尝试次数
}
```

### SmsProvider 适配

```typescript
/**
 * 短信渠道提供者 — 实现 VerificationChannelProvider 接口
 * 只负责"投递"验证码，不处理生成/校验/限流逻辑
 */
@Injectable()
export class SmsChannelProvider implements VerificationChannelProvider {
    readonly channel = 'sms';

    constructor(
        @Inject('SMS_PROVIDER') private readonly smsProvider: SmsProvider,
        private readonly sysConfig: SystemConfigService,
    ) {}

    async deliver(params: VerificationDeliverParams): Promise<VerificationDeliverResult> {
        const config = await this.sysConfig.getConfig<SmsProviderConfig>('sms.provider');
        const templateCode = config.templates[params.purpose as keyof typeof config.templates];

        const result = await this.smsProvider.send({
            phone: params.identifier,
            templateCode,
            templateParams: { code: params.code },
            signName: config.signName,
        });

        return {
            success: result.success,
            requestId: result.requestId,
            errorCode: result.errorCode,
            errorMessage: result.errorMessage,
        };
    }
}
```

### Redis Key 统一规范

| Key                                               | 类型          | TTL                  | 说明                            |
| ------------------------------------------------- | ------------- | -------------------- | ------------------------------- |
| `mono:verify:{channel}:{identifier}`              | STRING (JSON) | 可配置（默认 5 min） | 验证码 + 用途 + 渠道 + 尝试次数 |
| `mono:verify:{channel}:interval:{identifier}`     | STRING        | 可配置（默认 60 s）  | 发送间隔锁                      |
| `mono:verify:{channel}:daily:{identifier}:{date}` | INT           | 24 h                 | 同一标识每日发送计数            |
| `mono:verify:{channel}:ip:{ip}:{hour}`            | INT           | 1 h                  | 同一 IP 每小时发送计数          |

> 与之前 SmsService 中的 Redis Key 对比：新增了 `{channel}` 层级，短信和邮件的 key 互不干扰。
> 旧 key `mono:verify:sms:{phone}` → 新 key `mono:verify:sms:{phone}`（格式不变，只是逻辑上统一了）。

> **迁移说明**：原有的 `SmsService.sendVerificationCode()` 和 `SmsService.verifyCode()` 逻辑迁移到 `VerificationCodeService`，`SmsService` 只保留 `SmsChannelProvider.deliver()` 的投递逻辑。这是一个纯重构，不影响外部 API 接口。

## 十、安全注意事项

| 风险                        | 防护措施                                              |
| --------------------------- | ----------------------------------------------------- |
| 验证码暴力破解              | 5 次错误后作废，需重新发送                            |
| 短信轰炸（同一手机号）      | 60 秒间隔 + 每日 10 条上限                            |
| 短信轰炸（不同手机号同 IP） | IP 每小时 20 条上限                                   |
| 验证码重放                  | 一次性使用，验证通过即删除                            |
| 验证码用途篡改              | Redis 中存储 purpose，校验时比对                      |
| 手机号遍历                  | 注册/登录返回相同提示，不泄露"手机号是否已注册"       |
| 审计表拖库                  | 审计表不存明文验证码                                  |
| 密钥泄露                    | system_config 敏感字段 API 返回时脱敏，仅管理员可读写 |
| 配置篡改                    | system_config 写入需 admin 角色 + 操作审计            |

## 十一、driver 切换：mock ↔ aliyun

> 本项目默认在本地开发、E2E 测试、OAuth 集成期间使用 `mock` driver（验证码写入 `verification_code` 表 + 日志，不真发短信）。生产环境切换到 `aliyun` driver（或其他第三方）。两种 driver 都在 `system_config` 表维护，可随时切换。

**切换方法**：

```bash
# 1. 登录管理后台 → 配置中心 → 短信服务
# 2. 切换 driver：mock → aliyun
# 3. 填写 aliyun 子配置：accessKeyId、accessKeySecret、signName、templateCode
# 4. 提交 → 写入 system_config + 清 Redis `mono:data:system_config:sms.provider` 缓存
# 5. SmsModule 监听到 driver 变化 → 重新创建 AliyunProvider 实例
# 6. 下次发短信自动走 aliyun
```

**回滚**：

```bash
# 紧急回滚到 mock
pnpm -F @apps/server exec node -e "
  const { PrismaClient } = require('@prisma/client');
  const p = new PrismaClient();
  p.systemConfig.update({
    where: { key: 'sms.provider' },
    data: { value: { driver: 'mock', ... } },
  }).then(() => p.\$disconnect());
"
```

**注意**：

- 切换 driver 不需要重启服务（动态模块）
- 切换后 5 分钟内 pending 的验证码仍按切换前的 driver 处理（请求已开始）
- 审计日志会记录 driver 切换（`config_updated` action）
