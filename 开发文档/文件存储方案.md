# 文件存储方案

> 文件上传的两条路径：本地磁盘（开发/小项目）和云存储（生产/大流量）。通过统一接口切换，改一行配置即可。

---

## 概念速览

开始设计之前，先搞清楚几个名词。

### Multer 是什么

Multer 是 Node.js 的"文件接收器"。浏览器上传文件 → HTTP 请求到达服务器 → **Multer 拦截请求，把文件从网络流解析出来**，交给你处理。

```
浏览器 <form enctype="multipart/form-data">
  → POST /api/upload/avatar  { file: 二进制流 }
  → Multer 解析 multipart 请求
  → 你的代码拿到 req.file（文件对象）
  → 决定存哪里
```

没有 Multer，你只能自己手写二进制流解析，很痛苦。它内置在 `@nestjs/platform-express` 中，不需要额外安装。

### S3 是个协议，不是只有 AWS

**S3** 最早是 Amazon 的云存储产品（Simple Storage Service）。但因为用得太多，其他厂商纷纷实现同一套 API：

| 服务              | 谁家的     | 本质                                    |
| ----------------- | ---------- | --------------------------------------- |
| **AWS S3**        | Amazon     | 公有云对象存储，S3 协议的原版           |
| **MinIO**         | 开源社区   | 自己部署的 S3 兼容存储，Docker 一键启动 |
| **阿里云 OSS**    | 阿里云     | 公有云对象存储，新版支持 S3 协议        |
| **腾讯云 COS**    | 腾讯云     | 同上                                    |
| **Cloudflare R2** | Cloudflare | 同上（不收流量费）                      |

所以，只要你写的代码对接的是 **S3 协议**，上面这些服务**全部可以用**，改一下地址和密钥就行。

### 为什么要存"云"而不是"本地"

| 维度        | 本地磁盘                      | 云存储                       |
| ----------- | ----------------------------- | ---------------------------- |
| 文件丢不丢  | 服务器挂 → 文件丢             | 云厂商负责冗余备份           |
| 多台服务器  | 文件在 A 机器，B 机器访问不到 | 所有机器连同一个存储         |
| 带宽        | 用户下载占用服务器带宽        | 用户直接从云下载，不走服务器 |
| Docker 重启 | 文件消失（除非挂 volume）     | 不受影响                     |
| 成本        | 0（本地开发）                 | 按量付费，有免费额度         |

**简单结论**：本地开发用本地磁盘（方便），生产环境用云存储（可靠）。

## 架构设计

### 总体思路

```
       浏览器
          │
          ▼
   ┌─────────────┐
   │  NestJS      │  Multer 接收文件
   │  Controller  │
   └──────┬──────┘
          │ 调用接口（不关心底层谁实现）
          ▼
   ┌──────────────────┐
   │  IStorageService │  统一接口
   │  upload()        │
   │  getUrl()        │
   │  delete()        │
   │  getPresignedUrl()│
   └──────┬───────────┘
          │  运行时二选一
          │
   ┌──────┴──────┐
   │             │
   ▼             ▼
┌────────┐  ┌──────────────┐
│ Local  │  │     S3       │
│ 磁盘   │  │  (兼容协议)   │
└────────┘  └──────┬───────┘
                   │ 同一套代码，换配置
```

## 接口设计

所有实现都遵循同一个接口，业务代码不感知底层：

```typescript
// apps/server/src/storage/storage.interface.ts

/** 上传参数 */
export interface UploadOptions {
    /** 文件原始名（保留扩展名） */
    originalName: string;
    /** 文件二进制内容 */
    buffer: Buffer;
    /** MIME 类型（如 image/png） */
    mimeType: string;
    /** 存储子目录（如 avatars / documents） */
    folder: string;
}

/** 上传结果 */
export interface UploadResult {
    /** 存储后文件名（UUID 重命名后的） */
    storedName: string;
    /** 访问 URL */
    url: string;
    /** 文件大小（byte） */
    size: number;
}

/** 存储服务接口 — 所有实现都要满足 */
export interface IStorageService {
    /** 上传文件，返回访问地址 */
    upload(opts: UploadOptions): Promise<UploadResult>;

    /** 获取文件访问 URL */
    getUrl(storedName: string, folder: string): Promise<string>;

    /** 删除文件 */
    delete(storedName: string, folder: string): Promise<void>;

    /** 生成一次性上传凭证（前端直传用，不经过服务器） */
    getPresignedUrl(
        originalName: string,
        folder: string,
        mimeType: string,
    ): Promise<{ url: string; fields: Record<string, string> }>;
}
```

## 实现一：本地磁盘

最简单的实现。文件存在服务器本地目录，通过 NestJS 静态文件服务暴露 URL。

```typescript
// apps/server/src/storage/local-storage.service.ts
import { Injectable } from '@nestjs/common';
import { writeFile, unlink, mkdir } from 'node:fs/promises';
import { existsSync } from 'node:fs';
import { join, extname } from 'node:path';
import crypto from 'node:crypto';
import type { IStorageService, UploadOptions, UploadResult } from './storage.interface';

@Injectable()
export class LocalStorageService implements IStorageService {
    /** 文件存储根目录 */
    private readonly baseDir: string;

    /**
     * @param uploadDir  存储根目录（绝对路径），默认项目根目录下的 uploads/
     * @param baseUrl    拼接 URL 的前缀，如 http://localhost:3000/uploads
     */
    constructor(
        uploadDir: string = join(process.cwd(), 'uploads'),
        private readonly baseUrl: string = 'http://localhost:3000/uploads',
    ) {
        this.baseDir = uploadDir;
    }

    async upload(opts: UploadOptions): Promise<UploadResult> {
        // 1. UUID 重命名，保留原始扩展名
        const ext = extname(opts.originalName).toLowerCase();
        const storedName = `${crypto.randomUUID()}${ext}`;

        // 2. 确保目录存在
        const folderPath = join(this.baseDir, opts.folder);
        if (!existsSync(folderPath)) {
            await mkdir(folderPath, { recursive: true });
        }

        // 3. 写入磁盘
        const filePath = join(folderPath, storedName);
        await writeFile(filePath, opts.buffer);

        // 4. 拼接访问 URL
        const url = `${this.baseUrl}/${opts.folder}/${storedName}`;

        return { storedName, url, size: opts.buffer.length };
    }

    async getUrl(storedName: string, folder: string): Promise<string> {
        return `${this.baseUrl}/${folder}/${storedName}`;
    }

    async delete(storedName: string, folder: string): Promise<void> {
        const filePath = join(this.baseDir, folder, storedName);
        if (existsSync(filePath)) {
            await unlink(filePath);
        }
    }

    /**
     * 本地存储不支持 presigned URL —— 文件直接经服务器上传
     * 返回普通上传地址，前端走 <form> POST 到 NestJS
     */
    async getPresignedUrl(
        originalName: string,
        folder: string,
        mimeType: string,
    ): Promise<{ url: string; fields: Record<string, string> }> {
        // 本地模式：前端直接 POST 到服务器的 /api/upload/file 端点
        return {
            url: '/api/upload/file',
            fields: { folder, mimeType },
        };
    }
}
```

### 本地文件的 URL 访问

需要在 NestJS 中暴露静态文件目录：

```typescript
// apps/server/src/main.ts
import { NestFactory } from '@nestjs/core';
import { NestExpressApplication } from '@nestjs/platform-express';
import { join } from 'node:path';

async function bootstrap() {
    const app = await NestFactory.create<NestExpressApplication>(AppModule);

    // 让 /uploads/* 路径直接映射到磁盘上的 uploads 目录
    // 例如 http://localhost:3000/uploads/avatars/abc.jpg
    app.useStaticAssets(join(process.cwd(), 'uploads'), {
        prefix: '/uploads',
    });

    await app.listen(3000);
}
```

> ⚠️ **安全注意**：静态文件服务只应暴露 `uploads/` 目录，不要暴露项目根目录。生产环境应用 Nginx 替代（性能更好），NestJS 的静态服务仅用于本地开发。

## 实现二：S3 兼容协议

同一套代码对接 **MinIO、阿里云 OSS、AWS S3、腾讯云 COS、Cloudflare R2**。

核心依赖：`@aws-sdk/client-s3`（AWS 官方 SDK）。虽然名字里带 "AWS"，但它支持任何 S3 兼容的 endpoint。

```bash
npm install @aws-sdk/client-s3 @aws-sdk/s3-request-presigner
```

```typescript
// apps/server/src/storage/s3-storage.service.ts
import { Injectable } from '@nestjs/common';
import { S3Client, PutObjectCommand, GetObjectCommand, DeleteObjectCommand } from '@aws-sdk/client-s3';
import { getSignedUrl } from '@aws-sdk/s3-request-presigner';
import crypto from 'node:crypto';
import { extname } from 'node:path';
import type { IStorageService, UploadOptions, UploadResult } from './storage.interface';

export interface S3Config {
    /** S3 服务地址，如 http://localhost:9000（MinIO）或 https://oss-cn-hangzhou.aliyuncs.com */
    endpoint: string;
    /** 存储桶名 */
    bucket: string;
    /** 区域（MinIO 可以随便填如 us-east-1，OSS 填如 oss-cn-hangzhou） */
    region: string;
    /** Access Key */
    accessKeyId: string;
    /** Secret Key */
    secretAccessKey: string;
    /**
     * 是否使用路径寻址（Path-Style）
     * - true : http://endpoint/bucket/key（MinIO 用这个）
     * - false: http://bucket.endpoint/key（AWS S3 用这个）
     */
    forcePathStyle?: boolean;
    /** 公开访问的基础 URL（如 CDN 地址，不填则用 endpoint 拼接） */
    publicBaseUrl?: string;
}

@Injectable()
export class S3StorageService implements IStorageService {
    private readonly client: S3Client;
    private readonly config: S3Config;

    constructor(config: S3Config) {
        this.config = config;
        this.client = new S3Client({
            endpoint: config.endpoint,
            region: config.region,
            credentials: {
                accessKeyId: config.accessKeyId,
                secretAccessKey: config.secretAccessKey,
            },
            forcePathStyle: config.forcePathStyle ?? true, // 默认 Path-Style，兼容性最好
        });
    }

    async upload(opts: UploadOptions): Promise<UploadResult> {
        const ext = extname(opts.originalName).toLowerCase();
        const storedName = `${crypto.randomUUID()}${ext}`;
        const key = `${opts.folder}/${storedName}`;

        await this.client.send(
            new PutObjectCommand({
                Bucket: this.config.bucket,
                Key: key,
                Body: opts.buffer,
                ContentType: opts.mimeType,
            }),
        );

        const url = await this.getUrl(storedName, opts.folder);

        return { storedName, url, size: opts.buffer.length };
    }

    async getUrl(storedName: string, folder: string): Promise<string> {
        const key = `${folder}/${storedName}`;

        // 如果配置了 CDN / 自定义域名，优先使用
        if (this.config.publicBaseUrl) {
            return `${this.config.publicBaseUrl}/${key}`;
        }

        // 否则用 endpoint + bucket + key 拼接
        // Path-Style: http://endpoint/bucket/key
        // Virtual-Hosted: http://bucket.endpoint/key
        if (this.config.forcePathStyle) {
            return `${this.config.endpoint}/${this.config.bucket}/${key}`;
        }
        // 把 endpoint 的主机名替换为 bucket.endpoint 的格式
        const url = new URL(this.config.endpoint);
        url.hostname = `${this.config.bucket}.${url.hostname}`;
        url.pathname = `/${key}`;
        return url.toString();
    }

    async delete(storedName: string, folder: string): Promise<void> {
        const key = `${folder}/${storedName}`;
        await this.client.send(
            new DeleteObjectCommand({
                Bucket: this.config.bucket,
                Key: key,
            }),
        );
    }

    async getPresignedUrl(
        originalName: string,
        folder: string,
        mimeType: string,
    ): Promise<{ url: string; fields: Record<string, string> }> {
        const ext = extname(originalName).toLowerCase();
        const storedName = `${crypto.randomUUID()}${ext}`;
        const key = `${folder}/${storedName}`;

        // 生成一个有时效的上传签名 URL（5 分钟有效）
        const url = await getSignedUrl(
            this.client,
            new PutObjectCommand({
                Bucket: this.config.bucket,
                Key: key,
                ContentType: mimeType,
            }),
            { expiresIn: 300 },
        );

        // 前端对这个 URL 直接 PUT 文件即可，不走服务器
        return {
            url,
            fields: {
                key,
                'Content-Type': mimeType,
            },
        };
    }
}
```

### 各服务商的配置对照表

| 服务              | endpoint                                        | region              | forcePathStyle | 说明                      |
| ----------------- | ----------------------------------------------- | ------------------- | :------------: | ------------------------- |
| **MinIO**（本地） | `http://localhost:9000`                         | `us-east-1`（随意） |    ✅ true     | Docker 一键启动           |
| **阿里云 OSS**    | `https://oss-cn-hangzhou.aliyuncs.com`          | `oss-cn-hangzhou`   |    ❌ false    | region 填对应地域         |
| **AWS S3**        | `https://s3.amazonaws.com`                      | `us-east-1`         |    ❌ false    | forcePathStyle 建议 false |
| **腾讯云 COS**    | `https://cos.ap-guangzhou.myqcloud.com`         | `ap-guangzhou`      |    ❌ false    |                           |
| **Cloudflare R2** | `https://<account-id>.r2.cloudflarestorage.com` | `auto`              |    ✅ true     | 不收流量费                |

> **提示**：如果你不确定 `forcePathStyle` 填什么，先填 `true`（Path-Style）。这是最通用的方式，所有 S3 兼容服务都支持。

## 动态模块：根据配置自动选择

业务代码不需要知道用了哪种存储，模块初始化时自动选择：

```typescript
// apps/server/src/storage/storage.config.ts
import { registerAs } from '@nestjs/config';
import { z } from 'zod';

const storageConfigSchema = z.object({
    /** 存储模式：local = 本地磁盘，s3 = S3 兼容协议 */
    STORAGE_DRIVER: z.enum(['local', 's3']).default('local'),
    // local 模式需要的
    STORAGE_LOCAL_DIR: z.string().optional(),
    STORAGE_LOCAL_BASE_URL: z.string().optional(),
    // s3 模式需要的
    STORAGE_S3_ENDPOINT: z.string().optional(),
    STORAGE_S3_BUCKET: z.string().optional(),
    STORAGE_S3_REGION: z.string().optional(),
    STORAGE_S3_ACCESS_KEY: z.string().optional(),
    STORAGE_S3_SECRET_KEY: z.string().optional(),
    STORAGE_S3_FORCE_PATH_STYLE: z.string().optional(), // 环境变量传 "true" / "false"
    STORAGE_S3_PUBLIC_BASE_URL: z.string().optional(),
});

export default registerAs('storage', () => {
    const config = storageConfigSchema.parse(process.env);
    return config;
});
```

```typescript
// apps/server/src/storage/storage.module.ts
import { Module } from '@nestjs/common';
import type { ConfigType } from '@nestjs/config';
import storageConfig from './storage.config';
import { LocalStorageService } from './local-storage.service';
import { S3StorageService } from './s3-storage.service';
import type { IStorageService } from './storage.interface';

@Module({})
export class StorageModule {
    /**
     * 静态方法注册 — 根据 STORAGE_DRIVER 决定用哪个实现
     *
     * 用法：
     *   imports: [StorageModule.register()],
     */
    static register() {
        return {
            module: StorageModule,
            providers: [
                {
                    provide: 'STORAGE_SERVICE', // 注入 token
                    useFactory: (config: ConfigType<typeof storageConfig>) => {
                        if (config.STORAGE_DRIVER === 's3') {
                            return new S3StorageService({
                                endpoint: config.STORAGE_S3_ENDPOINT!,
                                bucket: config.STORAGE_S3_BUCKET!,
                                region: config.STORAGE_S3_REGION!,
                                accessKeyId: config.STORAGE_S3_ACCESS_KEY!,
                                secretAccessKey: config.STORAGE_S3_SECRET_KEY!,
                                forcePathStyle: config.STORAGE_S3_FORCE_PATH_STYLE === 'true',
                                publicBaseUrl: config.STORAGE_S3_PUBLIC_BASE_URL,
                            });
                        }
                        // 默认走本地
                        return new LocalStorageService(config.STORAGE_LOCAL_DIR, config.STORAGE_LOCAL_BASE_URL);
                    },
                    inject: [storageConfig.KEY],
                },
            ],
            exports: ['STORAGE_SERVICE'],
        };
    }
}
```

## 上传安全

不管用哪种存储，Controller 层的安全措施是一样的：

```typescript
// apps/server/src/upload/upload.controller.ts
import {
    Controller,
    Post,
    UseInterceptors,
    UploadedFile,
    ParseFilePipe,
    MaxFileSizeValidator,
    FileTypeValidator,
    UseGuards,
} from '@nestjs/common';
import { FileInterceptor } from '@nestjs/platform-express';
import { memoryStorage } from 'multer';
import { JwtAuthGuard } from '../common/guards/jwt-auth.guard';
import type { IStorageService } from '../storage/storage.interface';

@Controller('api/upload')
@UseGuards(JwtAuthGuard) // 只有登录用户能上传
export class UploadController {
    constructor(@Inject('STORAGE_SERVICE') private readonly storage: IStorageService) {}

    /**
     * 头像上传
     *
     * 安全链：
     *   1. JWT 认证（@UseGuards）
     *   2. 文件类型白名单（只允许 jpeg/png/webp）
     *   3. 文件大小限制（≤ 2MB）
     *   4. Multer memoryStorage（文件只在内存中，经 storage service 处理后即释放）
     *   5. UUID 重命名（storage service 内部，防路径遍历）
     */
    @Post('avatar')
    @UseInterceptors(
        FileInterceptor('file', {
            storage: memoryStorage(), // 存内存，不落盘
            limits: { fileSize: 2 * 1024 * 1024 }, // 2MB 硬限制
        }),
    )
    async uploadAvatar(
        @UploadedFile(
            new ParseFilePipe({
                validators: [
                    new MaxFileSizeValidator({ maxSize: 2 * 1024 * 1024 }),
                    new FileTypeValidator({ fileType: /^image\/(jpeg|png|webp)$/ }),
                ],
            }),
        )
        file: Express.Multer.File,
    ) {
        const result = await this.storage.upload({
            originalName: file.originalname,
            buffer: file.buffer,
            mimeType: file.mimetype,
            folder: 'avatars',
        });

        return { code: 0, data: result };
    }

    /**
     * 通用文件上传
     */
    @Post('file')
    @UseInterceptors(
        FileInterceptor('file', {
            storage: memoryStorage(),
            limits: { fileSize: 10 * 1024 * 1024 }, // 10MB
        }),
    )
    async uploadFile(
        @UploadedFile(
            new ParseFilePipe({
                validators: [new MaxFileSizeValidator({ maxSize: 10 * 1024 * 1024 })],
            }),
        )
        file: Express.Multer.File,
    ) {
        const result = await this.storage.upload({
            originalName: file.originalname,
            buffer: file.buffer,
            mimeType: file.mimetype,
            folder: 'files',
        });

        return { code: 0, data: result };
    }
}
```

> **为什么用 `memoryStorage()`**：文件存在内存中，不在服务器磁盘留痕迹。上传完成后 buffer 传给 storage service（本地模式写盘，S3 模式发网络请求），然后被垃圾回收。不推荐 `diskStorage()`——本地模式没问题，但切换到 S3 模式时，文件先写本地磁盘再读出来发 S3，多一次 I/O。

## 文件记录与后台管理

每次上传成功后，`upload.service.ts` 在数据库 `upload_file` 表中写一条记录：

```typescript
// apps/server/src/modules/upload/upload.service.ts
@Injectable()
export class UploadService {
    constructor(
        @Inject('STORAGE_SERVICE') private readonly storage: IStorageService,
        private readonly prisma: PrismaService,
    ) {}

    async upload(file: Express.Multer.File, accountId: string, folder: string) {
        // 1. 存储文件
        const result = await this.storage.upload({
            originalName: file.originalname,
            buffer: file.buffer,
            mimeType: file.mimetype,
            folder,
        });

        // 2. 写数据库记录
        await this.prisma.uploadFile.create({
            data: {
                accountId,
                originalName: file.originalname,
                storedName: result.storedName,
                mimeType: file.mimetype,
                size: result.size,
                storage: process.env.STORAGE_DRIVER === 's3' ? 's3' : 'local',
                folder,
                url: result.url,
            },
        });

        return result;
    }
}
```

有了数据库记录后，管理后台可以做：

| 功能         | 说明                                                                 |
| ------------ | -------------------------------------------------------------------- |
| **文件列表** | 表格展示所有文件：缩略图（图片类）、文件名、类型、大小、上传者、时间 |
| **筛选搜索** | 按文件类型（`mime_type` 前缀）、上传者、时间范围筛选                 |
| **删除文件** | 软删除（`deleted_at`），30 天后定时任务物理清除                      |
| **复制链接** | 一键复制文件访问 URL                                                 |
| **预览**     | 图片类在新标签页打开，其他类型下载                                   |

完整的 `upload_file` 表 DDL、GraphQL Query/Mutation 见 [数据库设计.md](./数据库设计.md#五文件管理1-张表)。

## 两条上传路径对比

```
路径 A：服务器中转（本地存储、小文件 S3）

  浏览器 → NestJS 服务器 → 本地磁盘
                      → S3（小文件可以，大文件吃内存）

路径 B：Presigned URL（生产推荐，大文件最佳）

  浏览器 → GET /api/upload/presigned-url  （拿到上传凭证）
         → PUT 直传到 S3/MinIO/OSS      （文件不经过 NestJS）
         → 浏览器回调 POST /api/upload/callback （通知服务器上传完成）
```

**选择标准**：

- 本地存储 → 只能用路径 A
- S3 + 头像/小文件（< 2MB）→ 路径 A，简单
- S3 + 大文件/高并发 → 路径 B，服务器零带宽

## 各环境配置速查

| 场景               | STORAGE_DRIVER | 额外组件                            | 适合                      |
| ------------------ | :------------: | ----------------------------------- | ------------------------- |
| 开发（最快启动）   |    `local`     | 无                                  | 本地写代码，不需要 Docker |
| 开发（模拟生产）   |      `s3`      | MinIO（docker compose up -d minio） | 验证 S3 上传逻辑          |
| 测试/CI            |    `local`     | 无                                  | CI 中没有 MinIO           |
| 生产（自建机房）   |      `s3`      | MinIO 集群                          | 数据不出企业              |
| 生产（阿里云）     |      `s3`      | OSS + CDN                           | 国内项目首选              |
| 生产（AWS）        |      `s3`      | S3 + CloudFront                     | 海外项目                  |
| 生产（Cloudflare） |      `s3`      | R2（不收流量费）                    | 出海项目，带宽成本敏感    |

## CDN 与图片处理

云存储的最佳实践是前面挂 CDN：

```
用户浏览器
    │
    ▼
  CDN（缓存 + 加速 + 图片处理）
    │
    ▼
  OSS / S3（源站，只被 CDN 回源访问，不直接暴露给用户）
```

对于头像场景，图片处理（裁剪/缩放/格式转换）在 CDN 层做比在后端做更好：

| 服务                | 图片处理方式                                                   |
| ------------------- | -------------------------------------------------------------- |
| 阿里云 OSS + CDN    | URL 后加参数：`?x-oss-process=image/resize,m_fill,w_200,h_200` |
| AWS S3 + CloudFront | Lambda@Edge 或 CloudFront Functions                            |
| Cloudflare R2       | Cloudflare Images（内置）                                      |

> 图片处理不在基座范围内实现。基座只管"把原文件存进去"，图片变体（缩略图、webp 格式等）由 CDN 或云服务 URL 参数处理。
