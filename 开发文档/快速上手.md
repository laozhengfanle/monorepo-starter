# 快速上手

> 5 分钟完成环境搭建，跑起整个项目。

## 1. 环境要求

| 工具    | 最低版本                  | 检查命令    |
| ------- | ------------------------- | ----------- |
| Node.js | ≥ 22 LTS                  | `node -v`   |
| pnpm    | ≥ 9.5（catalog 协议需要） | `pnpm -v`   |
| Docker  | Desktop 或 Engine         | `docker -v` |

启用 pnpm（Node.js 22 自带 corepack）：

```bash
corepack enable
corepack prepare pnpm@latest --activate
```

## 2. 一键启动

```bash
# 安装依赖
pnpm install

# 启动基础设施（PostgreSQL + Redis）
docker compose up -d

# 并行启动所有应用（server + admin + web）
pnpm dev
```

| 应用               | 地址                          |
| ------------------ | ----------------------------- |
| Server API         | http://localhost:3000         |
| GraphQL Playground | http://localhost:3000/graphql |
| Admin 后台         | http://localhost:5173         |
| Web 前端           | http://localhost:5174         |

## 3. 目录结构（30 秒版）

```
monorepo/
├── apps/
│   ├── server/     NestJS API（GraphQL + REST）
│   ├── admin/      管理后台（Vue 3）
│   └── web/        C端（Vue 3，镜像 admin 结构）
├── packages/
│   ├── shared/     共享 Zod schemas + 类型 + 工具
│   ├── config/     ESLint 共享配置
│   └── .husky/     Git hooks
└── docs/           设计文档
```

## 4. 常用命令

```bash
pnpm dev:server        # 只启动后端
pnpm dev:admin         # 只启动管理后台
pnpm dev:web           # 只启动 C端前端
pnpm -r lint           # 全仓代码检查
pnpm -r typecheck      # 全仓类型检查
pnpm -r test           # 全仓测试
```

## 5. 第一个 PR 流程

1. 了解你要改的功能对应的设计文档（`docs/` 目录下的 `.md` 文件）
2. 新建分支：`git checkout -b feat/your-feature`
3. 写代码 → `pnpm -r lint && pnpm -r typecheck && pnpm -r test`
4. 全部通过后 `git commit`（**禁止 `--no-verify`**，pre-commit hook 自动运行 prettier + eslint）
5. 发起 PR
