# 项目总览

## 定位：企业级基座

这是一个**可复用的企业级基座项目**。目标是把认证、鉴权、安全、日志、限流、缓存、错误码、文件上传等基础设施一次就位，后续新建项目 fork 这个基座即可直接开始写业务代码，不再重复搭架子。

三条设计原则：

1. **一次到位**：基础设施写到 production-ready 级别，不是 demo
2. **通用可复用**：不写死任何业务逻辑，所有模块可独立启用/禁用
3. **文档即交接**：docs/ 目录就是给下一个接手项目的人看的

## 仓库结构

pnpm workspace monorepo，`apps/` 放应用，`packages/` 放共享库。

```text
monorepo/
├── apps/
│   ├── admin/         管理后台前端（Vue 3）
│   ├── server/        NestJS 后端 API
│   └── web/           C端 前端
│
├── packages/
│   ├── config/        共享 ESLint 配置
│   └── shared/        共享工具函数、类型定义
│
└── docs/              设计文档
```

## 三个子项目

| 子项目 | 包名           | 说明                                                  |
| ------ | -------------- | ----------------------------------------------------- |
| admin  | `@apps/admin`  | 管理后台 SPA，供内部运营/管理员使用，Vue 3 + Naive UI |
| server | `@apps/server` | 统一 API 服务，NestJS，对接 admin 和 web 两个前端     |
| web    | `@apps/web`    | C端前端，面向消费者，Vue 3                            |

## 调用关系

```text
┌──────────┐     ┌──────────┐
│  admin   │     │   web    │   管理后台 + 用户端
└────┬─────┘     └────┬─────┘
     │  /graphql   │  /graphql       ← 数据 CRUD（GraphQL，95%+ 请求）
     │  /admin/auth/*  │  /auth/*             ← 认证（REST）
     ▼                 ▼
┌──────────────────────────┐
│         server           │   NestJS 统一 API
│   Query: me, adminAccounts, │   GraphQL（Apollo Driver）
│   Mutation: create...    │
│   REST: login, logout    │
└────────────┬─────────────┘
             │
             ▼
┌──────────────────────────┐
│    PostgreSQL + Redis     │
└──────────────────────────┘
```

admin 和 web 数据操作统一走 `/graphql`，认证走 REST。server 根据 JWT 中的 `userType` 区分请求来源，走不同的 Guard 和权限逻辑。

## 启动方式

```bash
pnpm install                # 安装所有依赖
pnpm dev                    # 并行启动 admin + web + server
pnpm dev:admin              # 单独启动管理后台 (port 5173)
pnpm dev:web                # 单独启动 C端前端 (port 5174)
pnpm dev:server             # 单独启动后端 (port 3000)
```
