# 04 配置参考

> **目标**：列出全部环境变量 / 配置文件 / 系统配置项，告诉你默认值、必填性、影响范围。
> 模板文件：
> - 后端：[`apps/server/.env.example`](../apps/server/.env.example)
> - 管理后台：[`apps/admin/.env.example`](../apps/admin/.env.example)
> - C 端：目前没有自定义环境变量

## 1. 后端环境变量

**位置**：`apps/server/.env`（git 忽略，运行时加载）

**12-Factor 原则**：所有配置走环境变量，**不要从 `NODE_ENV` 推断其他配置**。

### 数据库

| 变量 | 必填 | 默认值 | 说明 |
|---|---|---|---|
| `DATABASE_URL` | 是 | `postgresql://user:password@localhost:5433/mono_dev` | PostgreSQL 连接串，格式 `postgresql://USER:PASS@HOST:PORT/DB` |
| `DIRECT_URL` |  | （fallback 到 `DATABASE_URL`）| Prisma migrate 用的直连地址（生产环境用 PgBouncer 时区分） |

### Redis

| 变量 | 必填 | 默认值 | 说明 |
|---|---|---|---|
| `REDIS_URL` | 生产必填 | `redis://localhost:6379` | Redis 连接串。**开发环境不配置则降级为内存缓存**；**生产环境必须配** |

### JWT 认证

| 变量 | 必填 | 默认值 | 说明 |
|---|---|---|---|
| `JWT_SECRET` | 是 | dev 占位 | 32 字节随机 hex 字符串。生产用 `openssl rand -hex 32` |
| `JWT_ACCESS_TTL` |  | `900` | accessToken 有效期（秒），默认 15 分钟 |
| `JWT_REFRESH_TTL` |  | `604800` | refreshToken 有效期（秒），默认 7 天 |
| `JWT_ISSUER` |  | `monorepo-server` | JWT `iss` 字段 |
| `JWT_AUDIENCE` |  | `monorepo-app` | JWT `aud` 字段 |

### AES 加密

| 变量 | 必填 | 默认值 | 说明 |
|---|---|---|---|
| `AES_ENCRYPTION_KEY` | 是 | dev 占位（全 0）| 32 字节 hex 字符串。生产用 `openssl rand -hex 32`。**用于加密数据库里 OAuth token 等敏感字段** |

### 限流

| 变量 | 必填 | 默认值 | 说明 |
|---|---|---|---|
| `THROTTLE_LOGIN_TTL` |  | `900` | 登录失败限流窗口（秒），默认 15 分钟 |
| `THROTTLE_LOGIN_LIMIT` |  | `5` | 窗口内最多失败次数 |
| `THROTTLE_IP_LIMIT` |  | `50` | 单 IP 全局请求上限（次/秒） |

### 服务

| 变量 | 必填 | 默认值 | 说明 |
|---|---|---|---|
| `PORT` |  | `3000` | 服务监听端口 |
| `NODE_ENV` |  | `development` | `development` / `production` / `test`，影响异常响应是否含堆栈 |

### Cookie 安全

| 变量 | 必填 | 默认值 | 说明 |
|---|---|---|---|
| `COOKIE_SECURE` | 是 | `false` | accessToken / refreshToken cookie 的 Secure 标志。**生产必须 `true`**（HTTPS 才带 cookie） |
| `CSRF_COOKIE_SECURE` | 是 | `false` | csrf-token cookie 的 Secure 标志。**生产必须 `true`**（否则 `__Host-` 前缀 cookie 不下发） |

> **不要从 `NODE_ENV` 推断这两个值**——12-Factor 原则要求显式配置。

### CORS

| 变量 | 必填 | 默认值 | 说明 |
|---|---|---|---|
| `CORS_ORIGINS` | 生产必填 | dev fallback 到 `['http://localhost:5173','http://localhost:5174']` | 允许跨域的前端 origin，多个用逗号分隔。**生产环境未配则拒绝所有跨域请求** |

### 反向代理

| 变量 | 必填 | 默认值 | 说明 |
|---|---|---|---|
| `TRUSTED_PROXIES` |  | （空）| Express trust proxy 数值。`1` = 信任前 1 层代理（Nginx / ALB）。**直连部署必须留空** |

### 日志

| 变量 | 必填 | 默认值 | 说明 |
|---|---|---|---|
| `LOG_LEVEL` |  | `debug` | `debug` / `log` / `warn` / `error`。dev 用 `debug`，prod 用 `log` 或 `warn` |

### Prometheus 监控

| 变量 | 必填 | 默认值 | 说明 |
|---|---|---|---|
| `PROMETHEUS_ENABLED` |  | `false` | 是否暴露 `/metrics` 端点。生产默认关闭，需要时打开并配 `MetricsIpGuard` 限制内网访问 |

### 文件存储

| 变量 | 必填 | 默认值 | 说明 |
|---|---|---|---|
| `STORAGE_LOCAL_DIR` |  | `./uploads` | 本地存储目录（相对 server 启动目录） |
| `STORAGE_PUBLIC_BASE_URL` |  | `/uploads` | 文件公开访问 URL 前缀 |

### Turnstile（向后兼容）

| 变量 | 必填 | 默认值 | 说明 |
|---|---|---|---|
| `TURNSTILE_SITE_KEY` |  | （空）| Cloudflare Turnstile site key（前端用）。**主配置在 `system_config.turnstile.config`**，这里只是 fallback |
| `TURNSTILE_SECRET_KEY` |  | （空）| Cloudflare Turnstile secret（服务端校验用） |

> 留空等同于未配置，Turnstile 验证端点会跳过校验。

## 3. 系统配置（运行时，存数据库）

**位置**：`apps/admin` → 「系统设置」页面，底层表 `system_config`

**作用**：环境变量适合"启动期配置"（密钥、URL），`system_config` 适合"运行时可调配置"（driver 选择、限流阈值、UI 文案）。

**常用 key**（与 `apps/server/prisma/seed.ts` 同步）：

| key | 用途 | 示例 value |
|---|---|---|
| `settings` | 系统全局设置（密码策略、登录锁定、水印、请求超时） | `{"passwordMinLength": 8, "loginFailThreshold": 5, ...}` |
| `sms.provider` | SMS driver 切换 + 限流阈值 | `{"driver": "aliyun", "signName": "...", "limits": {...}}` |
| `mail.service` | Email driver 切换 + 限流阈值 | `{"driver": "mock", "from": {...}, "templates": {...}}` |
| `storage.driver` | 文件存储 driver | `{"driver": "local", "localDir": "./uploads", "baseUrl": "/uploads"}` |
| `oauth.providers` | 4 平台 OAuth（wechat-web / wechat-mp / wechat-miniprogram / apple） | `{"wechat-web": {"enabled": true, ...}, ...}` |
| `oauth.github` | GitHub OAuth 凭据 | `{"enabled": false, "clientId": "...", ...}` |
| `oauth.google` | Google OAuth 凭据 | `{"enabled": false, "clientId": "...", ...}` |
| `oauth.wechat` | 微信开放平台 OAuth 凭据 | `{"enabled": false, "appId": "...", ...}` |
| `turnstile.config` | Cloudflare Turnstile 配置 | `{"enabled": false, "siteKey": "...", "secretKey": "..."}` |
| `dashboard.quickEntries` | 仪表盘欢迎页快捷入口 | `[{"title": "...", "route": "..."}]` |

**修改方式**：

1. **管理后台 UI**：`/system/settings` 页面（推荐）
2. **直接改库**：`UPDATE system_config SET value = ... WHERE key = '...'`
3. **新增 key**：在管理后台「系统配置」页面手动加，或者写迁移脚本插入

**读不到 fallback**：读取时如果 key 不存在，返回 `null`，由调用方决定默认值。

## 4. 配置文件层级

环境变量优先级（从高到低）：

1. **OS 环境变量 / Docker -e / k8s env**：最高
2. **`apps/server/.env`**：本地开发
3. **`apps/server/.env.example` 中的默认值**：最低（仅作模板参考）

> 12-Factor 原则：环境变量是"真理之源"，`.env` 文件只是开发期便利，**生产环境绝不创建 `.env` 文件**，通过 Docker / k8s 注入。

## 5. 加新环境变量的步骤

举例：加一个 `FEATURE_NEW_BILLING` 开关

1. **改模板**：`apps/server/.env.example` 末尾追加（带注释说明）
2. **改 Zod 校验**：`apps/server/src/common/config/auth.config.ts`（或新建 `feature.config.ts`）用 Zod schema 校验
3. **注入 ConfigService**：
   ```ts
   constructor(private config: ConfigService) {
       this.enabled = this.config.get('FEATURE_NEW_BILLING') === 'true';
   }
   ```
4. **更新本表**：在本文档对应章节加一行
5. **更新 [08-Changelog.md](./08-Changelog.md)**：记录"加新环境变量 X"

## 6. 不要做的事

| 反模式 | 原因 |
|---|---|
| 从 `NODE_ENV === 'production'` 推断其他配置 | 违反 12-Factor，应显式配置 |
| 把密钥 commit 到 `.env.example` | `.env.example` 是公开模板，用占位值 |
| 硬编码 URL / 路径 | 一换环境就改代码 |
| 在 `system_config` 里存密钥 | 应该用环境变量；DB 备份会泄漏 |
| 在前端 `.env` 里放密钥 | 前端代码任何人可读，密钥必须放后端 |
