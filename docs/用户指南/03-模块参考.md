## Auth

**位置**：`apps/server/src/modules/auth/` + `apps/server/src/common/auth/`

**职责**：

- 登录（账号密码、短信验证码、第三方登录回调）
- 刷新 accessToken（双 token 机制：access 15min + refresh 7d）
- 登出（写 token 撤销记录）
- 当前用户信息（`me`）
- 修改密码

**关键 API（REST，前端直接调用）**：

| 端点 | 方法 | 用途 |
|---|---|---|
| `/api/auth/login` | POST | 账号密码登录 |
| `/api/auth/refresh` | POST | 刷新 accessToken |
| `/api/auth/logout` | POST | 登出（写黑名单） |
| `/api/auth/me` | GET | 当前用户信息 |
| `/api/auth/change-password` | POST | 改密 |
| `/api/auth/csrf-token` | GET | 预取 CSRF token |
| `/api/auth/send-sms` | POST | 发送短信验证码 |
| `/api/auth/send-email` | POST | 发送邮件验证码 |
| `/api/auth/reset-password/send` | POST | 发送重置密码验证码 |
| `/api/auth/reset-password` | POST | 用验证码重置密码 |

**关键 API（GraphQL）**：

```graphql
type Query {
  me: AccountType!           # 当前用户
}

type Mutation {
  updateMyProfile(input: UpdateMyProfileInput!): AccountType!
}
```

**鉴权装饰器**：

```ts
import { Public, Permission, RequireAuth, LoginOnly } from '@/common/decorators';

@Public()                       // 跳过 JwtAuthGuard（公开端点）
@Permission('admin:list')       // 要求 RBAC 权限码
@RequireAuth()                  // 必须登录但不查权限
@LoginOnly()                    // 只要求已登录，不查 RBAC
```

**Token 撤销机制**：

- `Account.tokenVersion` 字段：改密 / 软删账号时 +1，所有旧 token 失效
- `TokenRevocation` 表：精确撤销单个 jti（重置密码、登出）
- 双层防护：JWT payload.tokenVersion 校验 + 黑名单查询

**不能动的部分**：

- `JwtAuthGuard` 的 payload 校验逻辑（改了 token 撤销就失效）
- `auth.controller.ts` 的路由前缀 `/api/auth`
- `LoginLockService` 失败次数限流（防爆破）

## RBAC

**位置**：`apps/server/src/modules/admin/` + `apps/server/src/modules/member/`

**职责**：

- 管理端 RBAC：管理员 / 角色 / 菜单 / 权限码
- C 端 RBAC：用户 / 等级 / 菜单（与管理端镜像结构，互不影响）
- 权限缓存：基于 Redis 的两级缓存（角色级 + 账户级）
- 菜单树构建（前端侧边栏数据源）

**数据模型**（管理端）：

```text
admin_role              # 角色（code 是机器编码）
admin_menu              # 菜单（type: directory / page / button）
admin_account_role      # 账户↔角色
admin_role_menu         # 角色↔菜单
admin_account_menu      # 账户额外菜单（type: grant / deny）
```

**关键 API（GraphQL）**：

```graphql
# 角色
adminRoles(input: AdminRoleQueryInput!): AdminRoleConnection!
adminRole(id: ID!): AdminRole
createAdminRole(input: CreateAdminRoleInput!): AdminRole!
updateAdminRole(id: ID!, input: UpdateAdminRoleInput!): AdminRole!
deleteAdminRole(id: ID!): Boolean!
assignRoleMenus(roleId: ID!, input: AssignRoleMenusInput!): AdminRole!

# 菜单
adminMenus(input: AdminMenuQueryInput!): [AdminMenu!]!     # 返回树形
createAdminMenu(input: CreateAdminMenuInput!): AdminMenu!
updateAdminMenu(id: ID!, input: UpdateAdminMenuInput!): AdminMenu!
deleteAdminMenu(id: ID!): Boolean!

# 管理员
adminAccounts(input: AdminAccountQueryInput!): AdminAccountConnection!
createAdminAccount(input: CreateAdminAccountInput!): AdminAccount!
updateAdminAccount(id: ID!, input: UpdateAdminAccountInput!): AdminAccount!
deleteAdminAccount(id: ID!): Boolean!
assignAccountRoles(accountId: ID!, roleIds: [ID!]!): AdminAccount!
assignAccountMenus(accountId: ID!, input: AssignAccountMenusInput!): AdminAccount!

# 当前账户的菜单（侧边栏用）
myAdminMenuTree: [AdminMenu!]!
```

**权限码约定**：`资源:动作` 形式，例：

- `admin:list` / `admin:create` / `admin:update` / `admin:delete` / `admin:restore` / `admin:hard_delete`
- `role:list` / `role:assign-menus`
- `menu:list` / `menu:create`
- `dashboard:welcome` / `dashboard:analytics`（同一端点支持多权限码 OR 语义）

**前端权限指令**：

```vue
<button v-permission="'admin:create'">新增管理员</button>
```

**不能动的部分**：

- `AdminPermissionGuard` 校验流程（改了 RBAC 失效）
- `super_admin` 角色码（基座硬编码，删了会破坏默认账号 `root`）
- `aggregate-permissions.ts` 合并算法（grant / deny / 角色合并）

## CSRF

**位置**：`apps/server/src/common/middleware/csrf.middleware.ts`

**职责**：防止跨站请求伪造（Double Submit Cookie 模式）

**工作流**：

1. 登录响应 / 显式 GET `/api/auth/csrf-token` → 服务端下发 `csrf-token` cookie
2. 前端每次写请求（POST/PUT/DELETE/PATCH + GraphQL mutation）带 `X-CSRF-Token: <token>` 头
3. `csrfGuard` 中间件校验：header 必须等于 cookie

**前端封装**：`apps/admin/src/shared/request/csrf.ts` + `apps/web/src/api/csrf.ts` 已自动处理，前端代码不需要手动带 header。

**Cookie 命名规则**：

- 生产环境（`NODE_ENV=production`）：`__Host-csrf-token`（强制 HTTPS + 无 domain）
- 开发环境：`csrf-token`

**不能动的部分**：

- 中间件挂载顺序（`main.ts` 里必须在 cookieParser 之后、路由之前）
- `CSRF_COOKIE_SECURE` 必须显式配置（不要从 NODE_ENV 推断）
- `CORS_ORIGINS` 必须显式配置（生产环境未配则拒绝所有跨域）

## Throttler

**位置**：`apps/server/src/app.module.ts`（全局配置）+ `apps/server/src/common/guards/gql-aware-throttler.guard.ts`

**职责**：IP 级 / 登录端点 / GraphQL 限流

**三级限流（默认）**：

| 名称 | 窗口 | 限制 |
|---|---|---|
| `short` | 1 秒 | 3 次 |
| `medium` | 10 秒 | 20 次 |
| `long` | 60 秒 | 100 次 |

**登录端点单独限流**（`THROTTLE_LOGIN_TTL` + `THROTTLE_LOGIN_LIMIT`，默认 5 次/15 分钟）。

**`LoginLockService`**：登录失败次数达上限后锁定账号 N 分钟（防爆破）。

**不能动的部分**：

- `GqlAwareThrottlerGuard`（必须用这个，标准 `ThrottlerGuard` 在 GraphQL context 会抛 TypeError）
- 限流配置要走环境变量（不要硬编码）

## Cache

**位置**：`apps/server/src/common/cache/`

**职责**：

- Redis 缓存（主）
- 内存缓存降级（Redis 不可用时自动 fallback）
- 缓存 key 命名规范（`cache-key.constants.ts`）
- 账户权限缓存（角色级 + 账户级两级缓存）

**关键 API**：

```ts
import { CacheService } from '@/common/cache/cache.service';

await cache.get<T>('key');
await cache.set('key', value, ttlSeconds);
await cache.del('key');
await cache.delPattern('user:*');
```

**降级机制**：`RedisDegradationService.safeGet(key, fallback)` —— Redis 挂了返回 fallback 而不是抛错。

**不能动的部分**：

- `cache-key.constants.ts` 的 key 命名规范（改了会和旧数据冲突）
- `admin-permission-cache.service.ts` 的失效逻辑（改了会导致权限不更新）

## Audit

**位置**：`apps/server/src/common/audit/` + `apps/server/src/modules/audit/`

**职责**：记录敏感操作（登录、改密、权限变更、删除等），支持审计回溯

**关键 API**：

```ts
import { AuditService } from '@/modules/audit/audit.service';

await audit.log({
    action: 'admin.update',
    resourceType: 'admin',
    resourceId: adminId,
    detail: { changes: {...} },
});
```

**批量写入**：`AuditBatchService` 50 条/5s 批量入库，进程关闭时强制 flush，失败回滚到 `logs/audit-fallback.ndjson`。

**前端查询**：`apps/admin/src/features/system/logs/LogsPage.vue` 已有现成页面。

**不能动的部分**：

- `audit_log` 表结构（改了已有数据查不到）
- `AuditBatchService` 的 flush 触发条件（关了会丢日志）

## SMS / Email / OAuth

**位置**：

- `apps/server/src/common/sms/` —— 短信
- `apps/server/src/common/email/` —— 邮件
- `apps/server/src/common/oauth/` —— 第三方登录

**职责**：统一 provider 接口，支持运行时切换 driver

**当前内置 provider**：

| 模块 | Provider |
|---|---|
| SMS | `mock`（默认，本地开发用）、`aliyun`（阿里云） |
| Email | `mock`（默认） |
| OAuth | `wechat-web`（微信网页）、`wechat-mp`（微信公众号）、`wechat-miniprogram`（微信小程序）、`apple`（Apple 登录） |

**加新 provider**：复制 `mock.provider.ts` 实现接口，注册到对应 module 的 providers 数组，详见 [05-扩展指南.md](./05-扩展指南.md)。

## GraphQL

**位置**：`apps/server/src/modules/graphql/` + `apps/server/src/modules/<domain>/*.resolver.ts`

**协议**：Code-First（`@nestjs/graphql` 装饰器自动生成 schema）

**端点**：<http://localhost:3000/graphql>（Playground / Apollo Sandbox）

**Schema Artifact**：build 阶段生成 `apps/server/dist/schema.gql` 提交到 Git，供前端 codegen 消费。详见 `apps/server/docs/GraphQL.md`（位于 `docs/`，是开发文档）。

**关键约定**：

- 分页：`Connection<T>` 模式（cursor + edges + pageInfo）
- 输入：所有 mutation 用 Zod schema 校验（`@packages/shared`）
- 错误：业务错误通过 `extensions.code` 返回业务错误码（见下「错误码」）

**Depth Limit**：默认 10 层（防 DoS），可在 `graphql.module.ts` 调整。

**不能动的部分**：

- `graphql.module.ts` 的 introspection / playground 配置
- `schema.gql` artifact 必须每次 resolver 变更后重新生成（CI 会校验）

## 错误码

**位置**：`packages/shared/src/errors/error-codes.ts`

**约定**：1xxxx 通用、2xxxx 认证、3xxxx 验证码/短信/邮件、4xxxx OAuth

**完整列表**：

| 区间 | 含义 | 关键错误码 |
|---|---|---|
| 10001-10099 | 通用 | `INVALID_PARAMS` `NOT_FOUND` `CONFLICT` `INTERNAL_ERROR` |
| 20001-20099 | 认证 | `UNAUTHORIZED` `INVALID_CREDENTIALS` `INVALID_TOKEN` `LOGIN_LOCKED` |
| 30001-30099 | 短信 | `SMS_TOO_FREQUENT` `SMS_DAILY_LIMIT` `SMS_CODE_EXPIRED` `SMS_CODE_INVALID` |
| 30101-30199 | 邮件 | `EMAIL_TOO_FREQUENT` `EMAIL_CODE_EXPIRED` `EMAIL_CODE_INVALID` |
| 40001-40099 | OAuth | `OAUTH_STATE_INVALID` `OAUTH_BIND_CONFLICT` `OAUTH_LAST_IDENTITY` |

**前端使用**：

```ts
import { ERROR_CODES } from '@packages/shared';

if (error.extensions?.code === ERROR_CODES.SMS_CODE_INVALID) {
    // 提示用户验证码错误
}
```

**新增错误码**：在 ERROR_CODES 对象末尾追加（**不要插值**，会破坏已对接端），然后在 [08-Changelog.md](./08-Changelog.md) 记录。

## 前端共享资源

**位置**：

- `apps/admin/src/shared/` —— 管理后台复用

- `apps/*/src/shared/utils/` —— 工具

**Composables（常用）**：

| 名称 | 用途 |
|---|---|
| `useFormSubmit` | 表单提交（loading / 错误 / 重置） |
| `useMessage` | Naive UI 消息提示 |

| `useSystemInfo` | 系统信息（版本号等） |
| `useDesignTokens` | 设计 token（颜色、间距） |
| `usePersistedRef` | 持久化 ref 到 localStorage |

**Components（常用）**：

| 名称 | 用途 |
|---|---|
| `EmptyState` | 空状态占位 |
| `ErrorBoundary` | 错误边界 |
| `FieldRulePopover` | 表单字段聚焦时弹出校验规则 |

**请求层**：

- `graphql-client.ts` —— GraphQL 客户端（含 CSRF / 401 自动刷新 / 5xx 重试）

- `settings` —— 个性化设置（主题、布局）

**指令**：

- `v-permission="'admin:create'"` —— 权限指令
- `v-permission:any="['admin:create', 'admin:update']"` —— 任一权限即可
