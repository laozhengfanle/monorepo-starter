# MonoKit 技术文档

> **消费者文档** —— 面向 fork MonoKit 做二次开发的工程师。
> 与仓库内 `docs/` 的设计文档分开：本文档只讲 **怎么用**，不讲 **怎么来的**。

---

## 这是什么

MonoKit 是一个 **可复用的企业级全栈基座**（Starter Foundation），不是产品。

- **后端**：`apps/server` —— NestJS 11，GraphQL + REST 双协议
- **管理后台**：`apps/admin` —— Vue 3 + Naive UI
- **C 端**：`apps/web` —— Vue 3 + Naive UI
- **共享**：`packages/shared` —— Zod schema、ESLint/TS 配置

fork 这个仓库后，你应该 **删掉不需要的页面和模块，保留架构、认证、权限、缓存、日志这些基础设施**，然后开始写你的业务。

## 基线信息

| 项目 | 值 |
|---|---|
| 当前基座版本 | v1.2.0（npm 包名 `monorepo-starter`） |
| 协议 | MIT（详见仓库根 [LICENSE](../LICENSE)） |
| Node | ≥ 22 LTS |
| pnpm | 11.x（用 corepack 启用） |
| 运行时基础设施 | PostgreSQL 16、Redis 7 |

> **包名迁移说明**：仓库内 npm 包名仍为 `monorepo-starter`，对外品牌名为 MonoKit。fork 后第一步是把所有 `package.json` 的 `name` 字段改成你的项目命名空间，详见 [05-扩展指南.md](./05-扩展指南.md)。
