# 01 快速上手

> **目标**：15 分钟内把整个项目（后端 + 管理后台 + C 端）跑起来，看到默认登录页。

---

## 1. 环境要求

| 工具 | 最低版本 | 检查命令 |
|---|---|---|
| Node.js | ≥ 22 LTS | `node -v` |
| pnpm | ≥ 11（corepack 协议需要） | `pnpm -v` |
| Docker | Desktop 或 Engine | `docker -v` |
| Git | 任意 | `git --version` |

启用 pnpm（Node 22 自带 corepack）：

```bash
corepack enable
corepack prepare pnpm@latest --activate
```

---

## 2. 克隆与安装

```bash
# 克隆（替换为你的 fork 地址）
git clone <your-fork-url> my-project
cd my-project

# 安装所有依赖（apps/* + packages/*）
pnpm install
```

首次安装会下载 ~500MB 的 node_modules，需要 2-5 分钟。期间可以读 [02-架构总览.md](./02-架构总览.md)。

---

## 3. 启动基础设施

本项目用 PostgreSQL 16 + Redis 7，最快的方式是 Docker Compose：

```bash
# 启动 Postgres + Redis（后台运行，端口只绑 127.0.0.1）
docker compose up -d
```

启动后可以看到：

```text
NAME                IMAGE                PORTS
monorepo-postgres   postgres:16-alpine   127.0.0.1:5433->5432/tcp
monorepo-redis      redis:7-alpine       127.0.0.1:6379->6379/tcp
```

> **为什么端口是 5433 / 6379 而不是默认的 5432 / 6379？** 避免和你机器上已有的 Postgres / Redis 冲突。详见 `docker-compose.yml`。

---

## 4. 准备后端环境变量

```bash
# 复制模板
cp apps/server/.env.example apps/server/.env
```

打开 `apps/server/.env`，**至少改这两项**：

```bash
# 32 字节随机字符串（生产必填，本地用占位值即可）
JWT_SECRET=$(openssl rand -hex 32)

# 32 字节随机字符串（用于加密 OAuth token 等敏感字段，本地用占位值即可）
AES_ENCRYPTION_KEY=$(openssl rand -hex 32)
```

其他变量保持默认即可，详见 [04-配置参考.md](./04-配置参考.md)。

---

## 5. 初始化数据库

```bash
# 应用所有迁移（创建表 + 索引）
pnpm -F @apps/server db:migrate

# 插入种子数据（默认账号、角色、菜单、权限）
pnpm -F @apps/server db:seed
```

种子完成后会创建 **两个默认账号**：

| 账号类型 | 用户名 | 密码 | 用途 |
|---|---|---|---|
| 超级管理员 | `root` | `Root!123` | 管理后台登录，**首次登录后必须改密码** |
| C 端测试用户 | `zhangsan` | `Test!123` | C 端登录 |

---

## 6. 启动开发服务器

仓库根的 `pnpm dev` 会并行启动后端 + 管理后台 + C 端：

```bash
pnpm dev
```

第一次启动会先编译 `@packages/shared`（共享 Zod schema 库），等几秒后三个服务就绪：

| 服务 | 端口 | 访问地址 |
|---|---|---|
| 后端 API | 3000 | <http://localhost:3000> |
| GraphQL Playground | 3000 | <http://localhost:3000/graphql> |
| 管理后台 | 5173 | <http://localhost:5173> |
| C 端 | 5174 | <http://localhost:5174> |

如果只想要某一个：

```bash
pnpm dev:server     # 只启动后端
pnpm dev:admin      # 只启动管理后台
pnpm dev:web        # 只启动 C 端
```

---

## 7. 验证

打开 <http://localhost:5173>，用 `root` / `Root!123` 登录，能看到默认仪表盘 → 跑通。

打开 <http://localhost:5174>，用 `zhangsan` / `Test!123` 登录，能看到 C 端首页 → 跑通。

打开 <http://localhost:3000/health>，返回 `{"status":"ok"}` → 后端健康。

docker compose down -v
