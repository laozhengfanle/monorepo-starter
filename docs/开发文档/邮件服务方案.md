# 邮件服务方案

> 邮件是企业级基座的基础依赖：注册邮箱验证、密码重置、告警通知。本文定义服务商选型、API 封装、模板管理和降级策略。
> 设计原则：**服务商可替换、后台可配置**——与短信服务方案一致，管理后台修改 `system_config` 表即可切换服务商/密钥/模板，无需改代码、无需重启服务。

---

## 一、服务商选型

### 候选对比

| 维度         | Resend                  | 阿里云邮件推送 | SendGrid           |
| ------------ | ----------------------- | -------------- | ------------------ |
| 国内到达率   | ⭐⭐⭐（海外节点）      | ≥ 95%          | ⭐⭐⭐（海外节点） |
| 国际到达率   | ⭐⭐⭐⭐⭐              | ⭐⭐⭐         | ⭐⭐⭐⭐⭐         |
| 免费额度     | 3,000 封/月             | 200 封/天      | 100 封/天          |
| API 质量     | ⭐⭐⭐⭐⭐（现代 REST） | ⭐⭐⭐         | ⭐⭐⭐⭐           |
| 模板管理     | React Email / MJML      | 控制台 + API   | 动态模板 API       |
| Webhook      | 投递状态回调            | 投递状态回调   | 事件通知           |
| SDK          | 官方 Node.js SDK        | 官方 SDK       | 官方 Node.js SDK   |
| 定价（超出） | $20/50K 封              | ¥0.002/封      | $15/50K 封         |

### 推荐方案

**默认使用 Resend**，理由：

1. API 设计现代，SDK 轻量（`resend` npm 包），TypeScript 原生支持
2. 免费额度充足（3,000 封/月），适合基座项目初期
3. 支持 React Email / MJML 模板，开发体验好
4. Webhook 回调完善，可追踪投递状态

> 国内项目如需更高国内到达率，在管理后台修改 `system_config` 表中的 `email.provider` 配置切换为阿里云邮件推送，业务代码无需改动。

### system_config 配置结构

邮件相关配置存储在 `system_config` 表中，管理后台可动态修改：

| key                     | value 结构  | 说明                                     |
| ----------------------- | ----------- | ---------------------------------------- |
| `email.provider`        | 见下方 JSON | 邮件服务商主配置（driver、发件人、模板） |
| `email.provider.resend` | 见下方 JSON | Resend API 密钥（敏感，API 返回时脱敏）  |
| `email.provider.aliyun` | 见下方 JSON | 阿里云密钥（敏感，API 返回时脱敏）       |

```typescript
/** email.provider 配置结构 */
interface EmailProviderConfig {
    /** 当前使用的服务商：resend | aliyun | sendgrid */
    driver: string;
    /** 默认发件人地址 */
    from: {
        name: string; // 发件人名称，如 "Monorepo"
        email: string; // 发件人邮箱，如 "noreply@yourdomain.com"
    };
    /** 各用途对应的模板 ID 或模板名 */
    templates: {
        verify_email: string; // 邮箱验证
        reset_password: string; // 密码重置
        welcome: string; // 欢迎邮件
        alert_notification: string; // 告警通知
    };
    /** 频率限制参数 */
    limits: {
        interval: number; // 发送间隔（秒），默认 60
        daily: number; // 每日发送上限，默认 20
        ipHourly: number; // IP 每小时上限，默认 30
        codeTtl: number; // 验证码有效期（秒），默认 600（10 分钟）
        maxAttempts: number; // 验证码最大尝试次数，默认 5
    };
}

/** email.provider.resend 密钥配置 */
interface ResendCredential {
    apiKey: string;
}

/** email.provider.aliyun 密钥配置 */
interface AliyunEmailCredential {
    accessKeyId: string;
    accessKeySecret: string;
    region: string; // 如 hangzhou
}
```

### EmailService 业务层

```typescript
@Injectable()
export class EmailService {
    constructor(
        @Inject('EMAIL_PROVIDER') private readonly provider: EmailProvider,
        private readonly sysConfig: SystemConfigService,
        private readonly cache: ICacheService,
    ) {}

    /**
     * 发送验证码邮件
     * 1. 从 system_config 读取限流参数
     * 2. 检查发送频率限制
     * 3. 生成 6 位数字验证码
     * 4. 存入 Redis（TTL 由 system_config 配置）
     * 5. 从 system_config 读取模板，调用 EmailProvider 发送
     * 6. 写入审计表
     */
    async sendVerificationCode(email: string, purpose: EmailPurpose, ip: string): Promise<void> {
        // 从 system_config 读取配置
        const config = await this.sysConfig.getConfig<EmailProviderConfig>('email.provider');
        const { limits, from, templates } = config;

        // 频率限制检查（与短信服务逻辑一致）
        await this.checkRateLimit(email, ip, limits);

        // 生成验证码
        const code = this.generateCode();

        // 存入 Redis（key 命名与短信一致，前缀不同）
        const redisKey = `mono:verify:email:${email}`;
        await this.cache.setex(redisKey, limits.codeTtl, JSON.stringify({ code, purpose, attempts: 0 }));

        // 渲染邮件内容
        const subject = this.getSubject(purpose);
        const html = this.renderCodeTemplate(code, purpose, limits.codeTtl / 60);

        // 发送邮件
        const result = await this.provider.send({
            to: email,
            subject,
            html,
            templateId: templates[purpose],
            templateData: { code, ttl: String(Math.floor(limits.codeTtl / 60)) },
        });

        if (!result.success) {
            await this.cache.del(redisKey);
            throw new BusinessException(ErrorCode.EMAIL_SEND_FAILED, result.errorMessage);
        }

        // 设置发送间隔标记
        await this.cache.setex(`mono:verify:email:interval:${email}`, limits.interval, '1');

        // 审计记录
        await this.recordAudit(email, purpose, ip, result.messageId);
    }

    /**
     * 校验验证码（逻辑与短信验证码一致）
     * 返回 true 表示验证通过，同时删除验证码（一次性使用）
     */
    async verifyCode(email: string, code: string, purpose: EmailPurpose): Promise<boolean> {
        const config = await this.sysConfig.getConfig<EmailProviderConfig>('email.provider');
        const { maxAttempts } = config.limits;

        const redisKey = `mono:verify:email:${email}`;
        const stored = await this.cache.get(redisKey);

        if (!stored) {
            throw new BusinessException(ErrorCode.EMAIL_CODE_EXPIRED);
        }

        const data = JSON.parse(stored);

        if (data.purpose !== purpose) {
            throw new BusinessException(ErrorCode.EMAIL_CODE_PURPOSE_MISMATCH);
        }

        if (data.attempts >= maxAttempts) {
            await this.cache.del(redisKey);
            throw new BusinessException(ErrorCode.EMAIL_CODE_MAX_ATTEMPTS);
        }

        if (data.code !== code) {
            data.attempts += 1;
            await this.cache.setex(redisKey, await this.cache.ttl(redisKey), JSON.stringify(data));
            throw new BusinessException(ErrorCode.EMAIL_CODE_INVALID);
        }

        // 验证通过，删除验证码
        await this.cache.del(redisKey);
        return true;
    }

    /**
     * 发送通知邮件（非验证码类，如告警通知）
     */
    async sendNotification(to: string, subject: string, html: string): Promise<void> {
        const result = await this.provider.send({ to, subject, html });

        if (!result.success) {
            throw new BusinessException(ErrorCode.EMAIL_SEND_FAILED, result.errorMessage);
        }
    }

    /** 频率限制检查 */
    private async checkRateLimit(email: string, ip: string, limits: EmailProviderConfig['limits']): Promise<void> {
        // 发送间隔检查
        const intervalKey = `mono:verify:email:interval:${email}`;
        if (await this.cache.exists(intervalKey)) {
            throw new BusinessException(ErrorCode.EMAIL_SEND_TOO_FREQUENT);
        }

        // 同一邮箱每日上限
        const dailyKey = `mono:verify:email:daily:${email}:${this.getToday()}`;
        const dailyCount = await this.cache.incr(dailyKey);
        if (dailyCount === 1) {
            await this.cache.setex(dailyKey, 86400, 1);
        }
        if (dailyCount > limits.daily) {
            throw new BusinessException(ErrorCode.EMAIL_DAILY_LIMIT_EXCEEDED);
        }

        // 同一 IP 每小时上限
        const ipHourlyKey = `mono:verify:email:ip:${ip}:${this.getCurrentHour()}`;
        const ipCount = await this.cache.incr(ipHourlyKey);
        if (ipCount === 1) {
            await this.cache.setex(ipHourlyKey, 3600, 1);
        }
        if (ipCount > limits.ipHourly) {
            throw new BusinessException(ErrorCode.EMAIL_IP_LIMIT_EXCEEDED);
        }
    }

    /** 生成 6 位数字验证码 */
    private generateCode(): string {
        return crypto.randomInt(100000, 999999).toString();
    }

    /** 获取邮件主题 */
    private getSubject(purpose: EmailPurpose): string {
        const subjects: Record<EmailPurpose, string> = {
            verify_email: '邮箱验证',
            reset_password: '密码重置',
            bind_email: '绑定邮箱',
            welcome: '欢迎注册',
        };
        return subjects[purpose] || '验证码';
    }

    /** 渲染验证码邮件模板（简单 HTML，复杂模板建议用 React Email） */
    private renderCodeTemplate(code: string, purpose: string, ttlMinutes: number): string {
        return `
            <div style="max-width:600px;margin:0 auto;font-family:sans-serif;padding:20px;">
                <h2 style="color:#333;">${this.getSubject(purpose as EmailPurpose)}</h2>
                <p>您的验证码是：</p>
                <div style="font-size:32px;font-weight:bold;color:#18a058;letter-spacing:4px;padding:10px 0;">
                    ${code}
                </div>
                <p style="color:#666;">验证码 ${ttlMinutes} 分钟内有效，请勿泄露给他人。</p>
                <p style="color:#999;font-size:12px;">如非本人操作，请忽略此邮件。</p>
            </div>
        `;
    }

    /** 审计记录 */
    private async recordAudit(email: string, purpose: EmailPurpose, ip: string, messageId: string): Promise<void> {
        try {
            await this.prisma.verification_code.create({
                data: {
                    id: generateUuidV7(),
                    identifier: email,
                    code: '******',
                    purpose,
                    channel: 'email',
                    ip,
                    expires_at: new Date(Date.now() + 10 * 60 * 1000),
                },
            });
        } catch (error) {
            this.logger.warn(`Failed to write email audit record: ${error.message}`);
        }
    }

    private getToday(): string {
        return new Date().toISOString().slice(0, 10);
    }

    private getCurrentHour(): string {
        return new Date().toISOString().slice(0, 13);
    }
}

type EmailPurpose = 'verify_email' | 'reset_password' | 'bind_email' | 'welcome';
```

### EmailModule 动态模块

```typescript
@Module({})
export class EmailModule {
    /**
     * 从 system_config 表读取 email.provider.driver，动态注入对应实现
     */
    static forRoot(): DynamicModule {
        return {
            module: EmailModule,
            providers: [
                {
                    provide: 'EMAIL_PROVIDER',
                    useFactory: async (sysConfig: SystemConfigService) => {
                        const config = await sysConfig.getConfig<EmailProviderConfig>('email.provider');
                        const driver = config.driver;

                        const providerMap: Record<
                            string,
                            (config: EmailProviderConfig, sysConfig: SystemConfigService) => Promise<EmailProvider>
                        > = {
                            resend: async (cfg, sc) => {
                                const cred = await sc.getConfig<ResendCredential>('email.provider.resend');
                                return new ResendEmailProvider(cred.apiKey);
                            },
                            aliyun: async (cfg, sc) => {
                                const cred = await sc.getConfig<AliyunEmailCredential>('email.provider.aliyun');
                                return new AliyunEmailProvider(cred.accessKeyId, cred.accessKeySecret, cred.region);
                            },
                        };

                        const factory = providerMap[driver];
                        if (!factory) {
                            throw new Error(
                                `Unsupported email provider: ${driver}. Available: ${Object.keys(providerMap).join(', ')}`,
                            );
                        }

                        return factory(config, sysConfig);
                    },
                    inject: [SystemConfigService],
                },
                EmailService,
            ],
            exports: [EmailService],
        };
    }
}
```

### Resend 实现

```typescript
// common/email/providers/resend.provider.ts
import { Resend } from 'resend';
import type { EmailProvider, EmailSendParams, EmailSendResult } from '../email.provider';

export class ResendEmailProvider implements EmailProvider {
    private readonly client: Resend;

    constructor(apiKey: string) {
        this.client = new Resend(apiKey);
    }

    async send(params: EmailSendParams): Promise<EmailSendResult> {
        try {
            const { data, error } = await this.client.emails.send({
                from: params.templateId ? undefined : undefined, // 由 from 配置决定
                to: params.to,
                subject: params.subject,
                html: params.html,
                text: params.text,
            });

            if (error) {
                return {
                    success: false,
                    messageId: '',
                    errorCode: error.name,
                    errorMessage: error.message,
                };
            }

            return {
                success: true,
                messageId: data?.id || '',
            };
        } catch (err) {
            return {
                success: false,
                messageId: '',
                errorCode: 'RESEND_ERROR',
                errorMessage: err.message,
            };
        }
    }
}
```

## 三、验证码生命周期

与短信验证码逻辑完全一致，仅 key 前缀和渠道不同：

```text
发送请求
  │
  ├─ 从 system_config 读取限流参数
  │
  ├─❌ 频率限制检查失败 → 拒绝发送
  │
  ├─✅ 通过
  │   │
  │   ├─ 生成 6 位验证码
  │   ├─ 存入 Redis（TTL 由 system_config 配置，默认 10min）
  │   │   key:   mono:verify:email:{email}
  │   │   value: { code, purpose, attempts: 0 }
  │   │
  │   ├─ 从 system_config 读取模板
  │   ├─ 调用 EmailProvider.send()
  │   │   ├─❌ 发送失败 → 删除 Redis key → 返回错误
  │   │   └─✅ 发送成功 → 设置间隔标记
  │   │
  │   └─ 审计记录（写入 verification_code 表，channel='email'）
  │
验证请求
  │
  ├─ Redis key 不存在 → 验证码过期
  ├─ purpose 不匹配 → 用途错误
  ├─ attempts ≥ maxAttempts → 超过最大尝试次数
  ├─ code 不匹配 → attempts++
  └─✅ code 匹配 → 删除 key → 验证通过
```

### Redis Key 汇总

| Key                                      | 类型          | TTL                   | 说明                     |
| ---------------------------------------- | ------------- | --------------------- | ------------------------ |
| `mono:verify:email:{email}`              | STRING (JSON) | 可配置（默认 10 min） | 验证码 + 用途 + 尝试次数 |
| `mono:verify:email:interval:{email}`     | STRING        | 可配置（默认 60 s）   | 发送间隔锁               |
| `mono:verify:email:daily:{email}:{date}` | INT           | 24 h                  | 同一邮箱每日发送计数     |
| `mono:verify:email:ip:{ip}:{hour}`       | INT           | 1 h                   | 同一 IP 每小时发送计数   |

> 与短信服务的 key 命名规范一致：`mono:verify:{channel}:{entity}:{identifier}`。

## 五、降级策略

| 场景            | 影响           | 应对策略                                                              |
| --------------- | -------------- | --------------------------------------------------------------------- |
| 服务商 API 超时 | 邮件发不出去   | 1. 超时 5s 自动重试 1 次 2. 记录失败日志 3. 返回友好错误提示          |
| 服务商 API 宕机 | 邮件完全不可用 | 1. 返回"邮件服务暂不可用" 2. 建议用户稍后重试 3. 触发告警通知运维     |
| 邮件到达率低    | 用户收不到邮件 | 1. 检查域名 SPF/DKIM/DMARC 配置 2. 考虑切换服务商                     |
| 频率限制触发    | 正常用户被限   | 1. 提示"发送过于频繁" 2. 显示倒计时 3. 提供"重新发送"按钮（间隔后）   |
| DNS 解析失败    | 邮箱地址无效   | 1. 前端先做格式校验 2. 后端发送前可做 MX 记录检查（可选，有性能开销） |

## 六、DNS 配置

邮件发送前需配置域名 DNS 记录，否则会被接收方标记为垃圾邮件。

### 必配记录

| 记录类型 | 名称                | 值                                                  | 说明                     |
| -------- | ------------------- | --------------------------------------------------- | ------------------------ |
| SPF      | `@`                 | `v=spf1 include:resend.com ~all`                    | 声明允许发送邮件的服务器 |
| DKIM     | `resend._domainkey` | 由 Resend 提供                                      | 邮件签名验证             |
| DMARC    | `_dmarc`            | `v=DMARC1; p=none; rua=mailto:dmarc@yourdomain.com` | 邮件认证策略             |
| MX       | `@`                 | 由 Resend 提供（如需收件）                          | 邮件接收路由             |

> **Resend DNS 配置**：在 Resend 控制台 → Domains → 添加域名 → 按提示配置 DNS 记录 → 等待验证通过。

| 端点                                | 方法 | 说明           |
| ----------------------------------- | ---- | -------------- |
| `/member/auth/email/send`           | POST | 发送邮箱验证码 |
| `/member/auth/email/verify`         | POST | 验证邮箱验证码 |
| `/member/auth/email/reset-password` | POST | 邮箱密码重置   |
| `/member/auth/email/bind`           | POST | 绑定邮箱       |

### 请求/响应示例

```typescript
// POST /member/auth/email/send
{
    email: "user@example.com",
    purpose: "verify_email"  // verify_email | reset_password | bind_email
}

// POST /member/auth/email/verify
{
    email: "user@example.com",
    code: "123456",
    purpose: "verify_email"
}

// POST /member/auth/email/reset-password
{
    email: "user@example.com",
    code: "123456",
    newPassword: "NewPassword123!"
}
```

## 八、错误码

邮件相关错误码（与短信错误码对齐，前缀不同）：

| 错误码 | 含义                       |
| ------ | -------------------------- |
| 30100  | 邮件发送失败               |
| 30101  | 邮件验证码过期             |
| 30102  | 邮件验证码错误             |
| 30103  | 邮件验证码用途不匹配       |
| 30104  | 邮件验证码超过最大尝试次数 |
| 30105  | 邮件发送过于频繁           |
| 30106  | 邮件每日发送上限           |
| 30107  | 邮件 IP 每小时上限         |

> 错误码规范详见 [错误码.md](./错误码.md)。
