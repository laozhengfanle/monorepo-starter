# 目录结构

> 整个 monorepo 的目录树。所有文档中出现的目录示例均以此文为准，不再各自画树。

---

## 一、根目录

```text
monorepo/
├── apps/
│   ├── server/          → @apps/server     NestJS 后端 API（GraphQL + REST）
│   ├── admin/           → @apps/admin      管理后台 SPA（Vue 3）
│   └── web/             → @apps/web        C端 SPA（Vue 3）
│
├── packages/
│   ├── config/          → @packages/config  共享 ESLint 配置
│   └── shared/          → @packages/shared  共享 Zod schemas + 类型 + 工具函数
│
├── docs/                消费者技术文档（fork 后阅读）
├── 开发文档/             内部开发文档（设计决策依据）
│
├── pnpm-workspace.yaml
├── pnpm-lock.yaml
├── package.json          根 package.json（只放 devDependencies）
├── tsconfig.base.json    共享 TS 配置
└── .gitignore
```

### 目录规则

| 规则       | 说明                                                                                                               |
| ---------- | ------------------------------------------------------------------------------------------------------------------ |
| `common/`  | 只放基础设施（config、health、prisma、cache、storage、guard、pipe、filter、decorator、utils 等），**不放业务逻辑** |
| `modules/` | 按业务域拆分，每个域一个目录。域内文件：[详见下文](#文件命名规则)                                                  |
| `bff/`     | Backend For Frontend 聚合层，按端（admin/public）拆分，为前端页面定制 REST 端点                                    |
| `tasks/`   | @nestjs/schedule 定时任务                                                                                          |
| `metrics/` | Prometheus 指标采集与暴露                                                                                          |

### 文件命名规则

```text
{domain}.{type}.ts    — 域名单词
{domain}-{detail}.{type}.ts  — 需要区分时加细节

type 后缀：.module.ts / .controller.ts / .resolver.ts / .service.ts
          .guard.ts / .decorator.ts / .pipe.ts / .filter.ts
          .interceptor.ts / .interface.ts / .type.ts / .input.ts
          .config.ts / .schema.ts
```

| 文件            | 命名示例                                               |
| --------------- | ------------------------------------------------------ |
| Module          | `auth.module.ts`、`admin-account.module.ts`            |
| Controller      | `admin-auth.controller.ts`、`upload.controller.ts`     |
| Resolver        | `admin-account.resolver.ts`、`auth.resolver.ts`        |
| Service         | `auth.service.ts`、`admin-account.service.ts`          |
| Guard           | `jwt-auth.guard.ts`、`admin-permission.guard.ts`       |
| Decorator       | `permission.decorator.ts`、`require-auth.decorator.ts` |
| Pipe            | `zod-validation.pipe.ts`                               |
| Filter          | `global-exception.filter.ts`                           |
| Interceptor     | `logging.interceptor.ts`                               |
| Interface       | `storage.interface.ts`、`cache.interface.ts`           |
| Type (GraphQL)  | `admin-account.type.ts`、`auth.type.ts`                |
| Input (GraphQL) | `admin-account.input.ts`                               |
| Config          | `database.config.ts`、`storage.config.ts`              |

### 目录规则

| 规则                              | 说明                                                            |
| --------------------------------- | --------------------------------------------------------------- |
| `features/{domain}/`              | 一个业务域的所有代码放一起（组件 + API + 类型）                 |
| Feature 之间禁止互引              | `features/iam/admins/` 不能 import `features/iam/roles/` 的东西 |
| 两个以上 feature 用到 → `shared/` | 共享组件、composable、工具函数放这里                            |
| `app/`                            | 框架层配置（路由、布局），不放业务代码                          |
| `api/`                            | GraphQL 操作 + BFF REST 调用，与具体业务无关                    |

### 文件命名规则

| 类型         | 命名                      | 示例                                               |
| ------------ | ------------------------- | -------------------------------------------------- |
| 页面组件     | PascalCase + Page 后缀    | `LoginPage.vue`、`AdminsPage.vue`                  |
| 功能组件     | PascalCase                | `MenuFormDrawer.vue`、`AccountPermissionModal.vue` |
| 共享基础组件 | PascalCase（语义命名）    | `EmptyState.vue`、`SearchGrid.vue`                 |
| composable   | use + PascalCase          | `usePermission.ts`、`useFormSubmit.ts`             |
| API 文件     | 域名.ts（放 api/ 目录下） | `api/accounts.ts`                                  |
| 类型文件     | types.ts                  | `features/iam/menus/types.ts`                      |
| 路由         | index.ts + guard/ 目录    | `app/router/index.ts`                              |
